#운영체제 #교착상태 #회피 #필수 #운영체제/프로세스관리
## 정의
교착상태 회피를 위한 자원 할당 알고리즘, Banker's Algorithm
- 다익스트라가 제안한 알고리즘으로, 안전 상태(Safe State)를 유지하면서 자원을 할당하여 교착상태를 회피하는 기법
## 키워드
* 안전 상태(Safe State), 불안전 상태(Unsafe State), 안전 순서(Safe Sequence), Available, Max, Allocation, Need
## 암기법
* "가최할필" - 가용(Available), 최대요구(Max), 할당(Allocation), 필요(Need=Max-Allocation)
## 구성요소/특징/유형
| 구분 | 설명 | 비고 |
| ---- | ---- | ---- |
| Available | 현재 사용 가능 자원 수 | 시스템 보유 |
| Max | 프로세스별 최대 요구량 | 사전 선언 |
| Allocation | 현재 할당된 자원 수 | 진행 중 |
| Need | 추가 필요 자원(Max-Allocation) | 요청 가능량 |
