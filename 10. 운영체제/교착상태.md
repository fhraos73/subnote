#운영체제 #교착상태 #Deadlock #필수 #운영체제/프로세스관리
## 정의
둘 이상의 프로세스가 서로 자원을 기다리며 무한 대기하는 상태, Deadlock
- 다중 프로세스 환경에서 각 프로세스가 상대방이 점유한 자원을 요구하면서 무한히 대기하는 상태
## 키워드
* 상호배제, 점유대기, 비선점, 환형대기, 예방, 회피, 탐지, 회복
## 암기법
* "상점비환" - 상호배제, 점유대기, 비선점, 환형대기 (4대 필요조건)
* "예회탐복" - 예방, 회피, 탐지, 회복 (4대 해결기법)
## 구성요소/특징/유형
| 구분 | 설명 | 비고 |
| ---- | ---- | ---- |
| 상호배제 | 한 번에 하나만 자원 사용 | 필요조건 |
| 점유대기 | 자원 보유하며 추가 요청 | 필요조건 |
| 비선점 | 강제 회수 불가 | 필요조건 |

| 회피 | 안전 상태 유지 | 은행가 알고리즘 |
| 탐지 | 교착상태 발견 | 자원할당 그래프 |
| 회복 | 프로세스 종료/자원 선점 | 복구 비용 |
