#정보관리기술사 #보안 #개인정보 #가명화 #익명화 #정리

## 정의
개인정보 비식별 처리 기술, 가명화/익명화
- 가명화: 추가 정보 없이는 특정 개인을 알아볼 수 없도록 처리하는 것 (재식별 가능)
- 익명화: 더 이상 개인을 알아볼 수 없도록 처리하는 것 (재식별 불가능)

## 키워드
* 가명처리, 익명처리, k-익명성, l-다양성, t-근접성, 재식별, 적정성 평가

## 암기법
* k l t: (익명화 모델) k-익명성, l-다양성, t-근접성
* 가삭범총마: (가명처리 기법) 가명처리, 삭제, 범주화, 총계처리, 마스킹

## 특징
- 가명화: 추가 정보와 결합 시 재식별 가능, 개인정보로 취급
- 익명화: 재식별 불가능, 개인정보 아님
- 법적 차이: 가명정보는 제한적 활용, 익명정보는 자유 활용
- 적정성 평가: 비식별 수준 검증 필요

## 목적
- 개인정보 보호와 데이터 활용의 균형
- 빅데이터 분석, AI 학습 데이터 확보
- 법적 규제 준수 (개인정보보호법, GDPR)

## 구성요소
| 구분 | 기법 | 설명 |
|------|------|------|
| 가명처리 | 가명처리 | 식별자를 다른 값으로 대체 |
| 가명처리 | 마스킹 | 일부 값을 *로 대체 |
| 가명처리 | 범주화 | 값을 범위로 대체 (25세→20대) |
| 가명처리 | 총계처리 | 개별 값을 합계/평균으로 대체 |
| 가명처리 | 삭제 | 식별 가능 정보 삭제 |
| 익명화 모델 | k-익명성 | k명 이상 동일 속성 보장 |
| 익명화 모델 | l-다양성 | 민감 속성 l개 이상 다양성 |
| 익명화 모델 | t-근접성 | 민감 속성 분포 유사성 |
