#정보관리기술사 #보안 #접근제어 #AC #정리

## 정의
정보자산 접근 통제 기술, 접근제어(AC)
- 정보시스템의 자원에 대한 비인가된 접근을 감시하고, 접근을 요구하는 사용자를 식별하여 접근 허용 여부를 결정하는 보안 기술
- 인증(Authentication), 인가(Authorization), 감사(Audit)를 통한 체계적 접근 관리

## 키워드
* MAC, DAC, RBAC, ABAC, ACL, 주체, 객체, 접근권한, 최소권한

## 암기법
* MAC DAC RBAC ABAC: (접근제어 모델) 강제적, 임의적, 역할기반, 속성기반
  강임역속
* 인인감: (AAA) 인증, 인가, 감사

## 특징
- 주체/객체 기반: 접근 주체와 보호 객체 정의
- 정책 기반: 접근 제어 정책에 따른 권한 부여
- 최소 권한: 업무 수행에 필요한 최소 권한만 부여
- 감사 추적: 접근 이력 기록 및 모니터링

## 목적
- 정보자산의 기밀성, 무결성, 가용성 보장
- 비인가 접근 차단
- 내부 위협 방지 및 감사 추적

## 구성요소
| 구분  | 구성요소                                 | 설명                   |
| --- | ------------------------------------ | -------------------- |
| 주체  | 사용자/프로세스                             | 접근을 요청하는 능동적 개체      |
| 객체  | 파일/데이터/시스템                           | 접근 대상이 되는 수동적 개체     |
| 모델  | MAC(Mandatory Access Control)        | 강제적 접근제어, 보안 등급 기반   |
| 모델  | DAC(Discretionary Access Control)    | 임의적 접근제어, 소유자 권한 부여  |
| 모델  | RBAC(Role Based Access Control)      | 역할 기반 접근제어, 직무 역할 기반 |
| 모델  | ABAC(Attribute Based Access Control) | 속성 기반 접근제어, 동적 정책    |
| 구현  | ACL(Acess Control List)              | 접근 제어 목록, 객체 중심      |
| 구현  | CL(Capability List)                  | 능력 목록, 주체 중심         |

## 구성도
```
[접근제어 체계]
         │
    ┌────┴────┐
    ▼         ▼
[주체]     [객체]
(사용자)   (자원)
    │         │
    └────┬────┘
         ▼
   [접근제어 정책]
         │
    ┌────┼────┬────┐
    ▼    ▼    ▼    ▼
[MAC]  [DAC] [RBAC][ABAC]
(강제) (임의) (역할) (속성)
    │
    ▼
[접근 결정]
    │
    ├── 허용 (Grant)
    └── 거부 (Deny)
         │
         ▼
    [감사 로그]
```
