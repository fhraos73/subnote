#정보관리기술사 #보안 #해시함수 #무결성 #정리 #보안/암호화

## 정의
임의 길이 입력을 고정 길이 출력으로 변환하는 단방향 함수, 암호학적 해시 함수
- 역상저항, 제2역상저항, 충돌저항에 대해 안전성을 갖는 암호화 알고리즘
- 임의 길이 입력값에 대해 고정 길이의 해시값(Message Digest)을 생성하는 단방향 함수

## 키워드
* SHA-2, SHA-3, MD5, 단방향성, 충돌회피성, MAC, MDC, 역상저항

## 암기법
* 압효단충: (특징) 압축성, 효율성, 단방향성, 충돌회피성
* MAC MDC: (종류) MAC(비밀키 기반), MDC(키 없음)

## 특징
- 압축성: 임의 길이 입력 → 고정 길이 출력
- 효율성: 빠른 연산 속도
- 단방향성: 해시값으로 원본 복원 불가
- 충돌회피성: 동일 해시값 생성하는 두 입력 찾기 어려움

## 목적
- 데이터 무결성 검증
- 전자서명의 메시지 다이제스트 생성
- 패스워드 저장 및 검증
- 디지털 포렌식 증거 무결성 확인

## 구성요소
| 구분 | 구성요소 | 설명 |
|------|----------|------|
| 종류 | MAC | Message Authentication Code, 비밀키 사용 (CBC-MAC, HMAC) |
| 종류 | MDC | Modification Detection Code, 키 없음 (MD5, SHA) |
| 알고리즘 | MD5 | 128bit 출력, 취약점 발견으로 사용 비권장 |
| 알고리즘 | SHA-1 | 160bit 출력, 취약점 발견 |
| 알고리즘 | SHA-2 | SHA-256, SHA-384, SHA-512 등 |
| 알고리즘 | SHA-3 | Keccak 알고리즘 기반, 최신 표준 |

## 구성도
```
[임의 길이 메시지] ──▶ [해시 함수] ──▶ [고정 길이 해시값]
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
           [MDC]        [MAC]      [HMAC]
        (키 없음)     (비밀키)   (해시+비밀키)
              │            │            │
         ┌────┴────┐   ┌───┴───┐    ┌───┴───┐
         ▼         ▼   ▼       ▼    ▼       ▼
      [MD5]    [SHA-2] [CBC-MAC][MAA][HMAC-SHA1][HMAC-SHA256]
    (128bit)  (256bit)
              │
     ┌────────┴────────┐
     ▼                 ▼
[디지털 포렌식]   [전자서명]
(무결성 검증)    (메시지 다이제스트)
```
