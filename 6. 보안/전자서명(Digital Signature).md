#정보관리기술사 #보안 #전자서명 #인증 #정리 #보안/암호화

## 정의
공개키 기반 전자문서 서명 기술, 전자서명
- 서명자가 해당 전자문서에 서명하였음을 나타내기 위해 전자문서에 첨부되거나 논리적으로 결합된 전자적 형태의 정보
- 송신자 개인키로 서명, 송신자 공개키로 검증하여 인증, 무결성, 부인방지 보장

## 키워드
* 개인키, 공개키, 해시함수, PKI, 부인방지, 무결성, 인증, 전자봉투

## 암기법
* 송개송공: (절차) 송신자 개인키로 서명 → 송신자 공개키로 검증
* 위인부변재: (특징) 위조불가, 인증, 부인방지, 변경불가, 재사용불가

## 특징
- 위조 불가: 서명자만 서명 생성 가능
- 인증: 서명자 신원 확인
- 부인 방지: 서명 사실 부인 불가
- 변경 불가: 서명된 문서 변조 탐지
- 재사용 불가: 다른 문서에 서명 재사용 불가

## 목적
- 전자문서의 진정성 확인
- 송신자 인증 및 부인방지
- 데이터 무결성 검증

## 구성요소
| 구분 | 구성요소 | 설명 |
|------|----------|------|
| 서명 생성 | 해시 함수 | 메시지 다이제스트 생성 (SHA-256 등) |
| 서명 생성 | 개인키 | 해시값 암호화하여 서명 생성 |
| 서명 검증 | 공개키 | 서명 복호화하여 해시값 추출 |
| 서명 검증 | 해시 비교 | 원본 해시와 복호화 해시 비교 |
| 인프라 | PKI | 공개키 인증서 발급/관리 |
| 확장 | 전자봉투 | 전자서명 + 대칭키 암호화 결합 |

## 구성도
```
[서명 생성 - 송신자]
[원본 메시지] ──▶ [해시 함수] ──▶ [메시지 다이제스트]
                                        │
                              [송신자 개인키로 암호화]
                                        │
                                        ▼
                                  [전자서명]
                                        │
[원본 메시지 + 전자서명] ──────────────────────▶ [전송]

[서명 검증 - 수신자]
[전자서명] ──▶ [송신자 공개키로 복호화] ──▶ [해시값 A]
[원본 메시지] ──▶ [해시 함수] ──▶ [해시값 B]
                                        │
                              [해시값 A = 해시값 B ?]
                                        │
                              [서명 유효/무효 판정]
```
