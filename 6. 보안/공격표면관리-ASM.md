#정보관리기술사 #보안 #ASM #공격표면 #취약점관리 #정리

## 정의
외부 공격 노출 영역 관리, ASM(Attack Surface Management)
- 조직의 외부에 노출된 모든 디지털 자산과 잠재적 공격 진입점을 지속적으로 발견, 분류, 평가, 모니터링하는 보안 관리 프로세스
- 공격자 관점에서 조직의 취약점을 파악하고 관리

## 키워드
* 공격 표면, 외부 자산, 취약점, 지속적 모니터링, EASM, CAASM

## 암기법
* 발분평모: (프로세스) 발견, 분류, 평가, 모니터링

## 특징
- 외부 관점: 공격자 시각에서 노출 자산 파악
- 지속적 관리: 실시간 자산 변화 추적
- 자동화: 자동 스캔 및 분석
- 통합 관리: 분산된 자산의 통합 가시성

## 목적
- 알려지지 않은 외부 노출 자산 발견
- 공격 표면 최소화
- 취약점 우선순위 기반 대응

## 구성요소
| 구분 | 구성요소 | 설명 |
|------|----------|------|
| 프로세스 | 자산 발견 | 외부 노출 자산 자동 탐지 |
| 프로세스 | 자산 분류 | 자산 유형, 중요도 분류 |
| 프로세스 | 취약점 평가 | 취약점 식별 및 위험 평가 |
| 프로세스 | 지속 모니터링 | 실시간 변화 감시 |
| 대상 | 도메인/서브도메인 | DNS 기반 자산 |
| 대상 | IP 주소 | 외부 노출 IP |
| 대상 | 웹 애플리케이션 | 웹 서비스, API |
| 대상 | 클라우드 자산 | 클라우드 인스턴스, 스토리지 |
| 유형 | EASM | 외부 공격 표면 관리 |
| 유형 | CAASM | 사이버 자산 공격 표면 관리 |

## 구성도
```
[ASM 프로세스]

[1. 자산 발견]
(Discovery)
      │
      ▼
[외부 노출 자산 탐지]
- 도메인/서브도메인
- IP 주소
- 웹 애플리케이션
- 클라우드 자산
- 인증서
      │
      ▼
[2. 자산 분류]
(Inventory)
      │
      ▼
[자산 유형/중요도 분류]
- 소유권 확인
- 비즈니스 연관성
      │
      ▼
[3. 취약점 평가]
(Assessment)
      │
      ▼
[취약점 식별 및 우선순위]
- 취약점 스캔
- 위험 점수 산정
      │
      ▼
[4. 지속 모니터링]
(Monitoring)
      │
      ▼
[실시간 변화 감시]
- 신규 자산 탐지
- 설정 변경 감지
- 취약점 알림

[ASM vs 기존 취약점 관리]
┌─────────────────┬─────────────────┐
│   기존 취약점 관리  │       ASM       │
├─────────────────┼─────────────────┤
│ 알려진 자산 대상   │ 미지 자산 포함   │
│ 주기적 스캔       │ 지속적 모니터링  │
│ 내부 관점        │ 외부(공격자) 관점│
└─────────────────┴─────────────────┘
```
