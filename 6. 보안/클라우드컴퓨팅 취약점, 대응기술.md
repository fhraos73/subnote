#정보관리기술사 #보안 #클라우드 #취약점 #정리 #보안/IoT/신기술

## 정의
클라우드 환경 보안 위협 및 대응, 클라우드컴퓨팅 취약점
- 클라우드 컴퓨팅 환경에서 발생할 수 있는 보안 취약점과 이에 대한 기술적, 관리적 대응 방안
- 가상화, 멀티테넌시, 데이터 집중화 등 클라우드 특성에 따른 고유 보안 위협

## 키워드
* 가상화 취약점, 멀티테넌시, 데이터 유출, API 보안, 계정 탈취, CSA

## 암기법
* 데계내 불공가: (CSA 위협) 데이터 유출, 계정 탈취, 악성 내부자, 불안전한 API, 공유 기술, 가상화

## 특징
- 공유 자원: 멀티테넌시로 인한 격리 문제
- 가상화 의존: 하이퍼바이저 취약점 영향
- 데이터 집중: 대규모 데이터 유출 위험
- 책임 공유: CSP와 사용자 간 책임 분담

## 목적
- 클라우드 환경의 보안 위협 식별
- 효과적인 보안 대책 수립
- 안전한 클라우드 서비스 운영

## 구성요소
| 구분 | 취약점 | 대응기술 |
|------|--------|----------|
| 데이터 | 데이터 유출 | 암호화, DLP, 접근제어 |
| 데이터 | 데이터 손실 | 백업, 복제, DR |
| 계정 | 계정 탈취 | MFA, IAM, 모니터링 |
| 계정 | 권한 상승 | 최소 권한, RBAC |
| API | 불안전한 API | API 게이트웨이, 인증 |
| 가상화 | VM 탈출 | 하이퍼바이저 패치, 격리 |
| 가상화 | 사이드채널 | 물리적 격리, 암호화 |
| 네트워크 | 네트워크 공격 | 방화벽, IDS/IPS, 세그멘테이션 |
| 관리 | 설정 오류 | CSPM, 자동화 점검 |
| 규정 | 컴플라이언스 | 인증(CSAP), 감사 |
