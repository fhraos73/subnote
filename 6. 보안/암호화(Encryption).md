#정보관리기술사 #보안 #암호화 #정리 #보안/암호화

## 정의
정보의 기밀성 보장을 위한 데이터 변환 기술, 암호화
- 평문(Plaintext)을 암호문(Ciphertext)으로 변환하여 인가되지 않은 사용자가 정보를 읽을 수 없도록 하는 기술
- 인증(Authentication), 기밀성(Confidentiality), 무결성(Integrity), 부인방지(Non-Repudiation) 보장

## 키워드
* 대칭키, 비대칭키, 해시함수, 혼돈(Confusion), 확산(Diffusion), 블록암호, 스트림암호

## 암기법
* 대체블치확혼확압: (원리) 대체, 블록, 치환, 확장, 혼돈, 확산, 압축
* 이소타전: (공개키 유형) 이산대수기반, 소인수분해기반, 타원곡선방식, 전자서명방식
* 스블: (대칭키 유형) 스트림암호, 블록암호

## 특징
- 기밀성: 인가되지 않은 사용자의 정보 접근 방지
- 무결성: 데이터 변조 여부 확인 가능
- 인증: 송신자 신원 확인
- 부인방지: 송신 사실 부인 불가

## 목적
- 정보의 기밀성(Confidentiality) 보장
- 데이터 무결성(Integrity) 검증
- 사용자 인증(Authentication) 및 부인방지(Non-Repudiation)

## 구성요소
| 구분   | 구성요소            | 설명                    |
| ---- | --------------- | --------------------- |
| 입력   | 평문(Plaintext)   | 암호화 대상 원본 데이터         |
| 알고리즘 | 암호화 알고리즘        | 평문을 암호문으로 변환하는 수학적 함수 |
| 키    | 암호키(Key)        | 암호화/복호화에 사용되는 비밀 값    |
| 출력   | 암호문(Ciphertext) | 암호화된 결과 데이터           |
| 방식   | 대칭키/비대칭키        | 키 사용 방식에 따른 분류        |

## 구성도
```
[평문] ──▶ [암호화 알고리즘] ──▶ [암호문]
                 │
            [암호키]
                 │
     ┌───────────┴───────────┐
     ▼                       ▼
[대칭키 암호화]        [비대칭키 암호화]
(DES, AES, SEED)      (RSA, ECC, ElGamal)
     │                       │
     ▼                       ▼
[스트림암호]  [블록암호]  [공개키]  [개인키]
(RC4)       (AES)      (암호화)  (복호화)
```
