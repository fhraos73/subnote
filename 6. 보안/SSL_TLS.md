#정보관리기술사 #보안 #SSL #TLS #암호화통신 #정리 #보안/인증/접근제어

## 정의
전송 계층 보안 프로토콜, SSL/TLS
- Secure Sockets Layer(SSL) / Transport Layer Security(TLS)는 네트워크 통신에서 데이터의 기밀성, 무결성, 인증을 보장하는 암호화 프로토콜
- 클라이언트-서버 간 안전한 통신 채널을 구축하는 표준 보안 프로토콜

## 키워드
* 핸드쉐이크, 인증서, 대칭키, 비대칭키, HTTPS, 세션키, CA

## 암기법
* 핸암경레: (프로토콜 구성) 핸드쉐이크, 암호사양변경, 경고, 레코드
* 클서키암: (핸드쉐이크) 클라이언트Hello, 서버Hello, 키교환, 암호화통신

## 특징
- 기밀성: 대칭키 암호화로 데이터 보호
- 무결성: MAC으로 데이터 변조 탐지
- 인증: 인증서 기반 서버/클라이언트 인증
- 계층 독립: 응용 계층과 전송 계층 사이 동작

## 목적
- 네트워크 통신의 도청 방지
- 데이터 무결성 보장
- 통신 상대방 인증

## 구성요소
| 구분 | 구성요소 | 설명 |
|------|----------|------|
| 프로토콜 | 핸드쉐이크 | 인증, 키 교환, 암호 협상 |
| 프로토콜 | 암호사양변경 | 암호 스위트 변경 알림 |
| 프로토콜 | 경고 | 오류 및 경고 메시지 |
| 프로토콜 | 레코드 | 데이터 분할, 압축, 암호화 |
| 인증 | CA 인증서 | 서버 신원 인증 |
| 키 교환 | RSA | RSA 공개키로 키 교환 |
| 키 교환 | DHE/ECDHE | Diffie-Hellman 키 교환 |
| 암호화 | AES | 대칭키 암호화 |
| 무결성 | HMAC | 메시지 인증 코드 |

## 구성도
```
[SSL/TLS 핸드쉐이크]

[Client]                              [Server]
    │                                     │
    │──── Client Hello ──────────────────▶│
    │     (버전, 암호스위트, 난수)          │
    │                                     │
    │◀─── Server Hello ───────────────────│
    │     (선택된 암호스위트, 난수)         │
    │◀─── Certificate ────────────────────│
    │     (서버 인증서)                    │
    │◀─── Server Hello Done ──────────────│
    │                                     │
    │──── Client Key Exchange ───────────▶│
    │     (Pre-Master Secret)             │
    │──── Change Cipher Spec ────────────▶│
    │──── Finished ──────────────────────▶│
    │                                     │
    │◀─── Change Cipher Spec ─────────────│
    │◀─── Finished ───────────────────────│
    │                                     │
    │◀════ 암호화된 데이터 통신 ═══════════▶│

[TLS 버전]
TLS 1.0 → TLS 1.1 → TLS 1.2 → TLS 1.3
(취약)    (취약)    (권장)    (최신)
```
