#정보관리기술사 #보안 #암호화 #RSA #옵션 #정리

## 정의
소인수분해 기반 공개키 암호화 알고리즘, RSA
- Rivest, Shamir, Adleman이 개발한 소인수분해의 어려움에 기반한 공개키 암호화 알고리즘
- 암호화와 전자서명 모두 지원하는 대표적인 비대칭키 암호화 방식

## 키워드
* 소인수분해, 공개키, 개인키, 모듈러 연산, 전자서명, 키 교환

## 암기법
* 소공개전: (특징) 소인수분해, 공개키/개인키, 전자서명

## 특징
- 소인수분해 기반: 큰 소수의 곱을 인수분해하기 어려운 점 이용
- 양방향 암호화: 공개키 암호화/개인키 복호화, 개인키 서명/공개키 검증
- 키 길이: 2048bit 이상 권장
- 속도: 대칭키 대비 느림, 키 교환/서명에 주로 사용

## 목적
- 안전한 키 교환
- 전자서명을 통한 인증 및 부인방지
- 기밀 데이터 암호화

## 구성요소
| 구분 | 구성요소 | 설명 |
|------|----------|------|
| 키 생성 | p, q | 두 개의 큰 소수 선택 |
| 키 생성 | n = p × q | 모듈러스 계산 |
| 키 생성 | φ(n) = (p-1)(q-1) | 오일러 함수 |
| 공개키 | (e, n) | e: 1 < e < φ(n), gcd(e, φ(n)) = 1 |
| 개인키 | (d, n) | d × e ≡ 1 (mod φ(n)) |
| 암호화 | c = m^e mod n | 평문 m을 암호문 c로 변환 |
| 복호화 | m = c^d mod n | 암호문 c를 평문 m으로 복원 |

## 구성도
```
[키 생성]
    │
    ▼
[소수 p, q 선택] ──▶ [n = p × q]
    │                     │
    ▼                     ▼
[φ(n) = (p-1)(q-1)]  [공개키 (e, n)]
    │                     │
    ▼                     ▼
[개인키 (d, n)]      [암호화: c = m^e mod n]
    │                     │
    ▼                     ▼
[복호화: m = c^d mod n]  [서명: s = m^d mod n]
    │                     │
    └─────────┬───────────┘
              ▼
    [전자서명/암호화 활용]
```
