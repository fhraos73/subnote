#정보관리기술사 #보안 #스마트카 #커넥티드카 #자율주행 #정리

## 정의
커넥티드/자율주행 차량 보안, 스마트카
- 네트워크에 연결되어 다양한 서비스를 제공하는 커넥티드카와 자율주행 차량의 보안 위협 및 대응 방안
- V2X 통신, ECU, CAN 버스 등 차량 시스템의 사이버 보안

## 키워드
* V2X, CAN 버스, ECU, OBD-II, 원격 제어, 자율주행, ISO/SAE 21434

## 암기법
* V2X: (통신) V2V(차량-차량), V2I(차량-인프라), V2P(차량-보행자), V2N(차량-네트워크)

## 특징
- 네트워크 연결: 외부 공격 표면 확대
- 복잡한 시스템: 다수의 ECU, 통신 프로토콜
- 물리적 위험: 사이버 공격이 인명 피해로 연결
- 실시간 요구: 지연 없는 보안 처리 필요

## 목적
- 스마트카 보안 위협 식별
- 안전한 커넥티드/자율주행 환경 구축
- 차량 사이버 보안 표준 준수

## 구성요소
| 구분 | 취약점 | 대응방안 |
|------|--------|----------|
| 통신 | V2X 통신 가로채기 | PKI 기반 인증, 암호화 |
| 통신 | 원격 접속 취약점 | 방화벽, 접근제어 |
| 내부 | CAN 버스 취약점 | 메시지 인증, 암호화 |
| 내부 | ECU 취약점 | 보안 부팅, 펌웨어 서명 |
| 인터페이스 | OBD-II 취약점 | 물리적 보안, 인증 |
| 인터페이스 | 인포테인먼트 | 망분리, 샌드박싱 |
| 센서 | 센서 스푸핑 | 센서 퓨전, 이상 탐지 |
| 업데이트 | OTA 취약점 | 서명 검증, 암호화 |

## 구성도
```
[스마트카 보안 아키텍처]

[외부 연결]
    │
    ├── V2X 통신 (V2V, V2I, V2P, V2N)
    ├── 텔레매틱스 (4G/5G)
    ├── Wi-Fi/Bluetooth
    └── OBD-II
    │
[게이트웨이 ECU] ◀── 방화벽, IDS
    │
[내부 네트워크]
    │
    ├── [파워트레인 도메인]
    │   (엔진 ECU, 변속기 ECU)
    │
    ├── [섀시 도메인]
    │   (브레이크 ECU, 조향 ECU)
    │
    ├── [ADAS 도메인]
    │   (카메라, 레이더, 라이다)
    │
    └── [인포테인먼트 도메인]
        (IVI, 내비게이션)

[보안 대응]
    │
    ├── V2X PKI (인증서 기반 인증)
    ├── SecOC (CAN 메시지 인증)
    ├── HSM (하드웨어 보안 모듈)
    ├── 보안 부팅 (Secure Boot)
    └── ISO/SAE 21434 준수
```
