#정보관리기술사 #보안 #IoT #취약점 #옵션 #정리

## 정의
사물인터넷 환경 보안 위협, IoT 보안 위협
- IoT(Internet of Things) 디바이스 및 네트워크에서 발생할 수 있는 보안 취약점과 공격 위협
- 제한된 리소스, 다양한 프로토콜, 대규모 연결 등 IoT 특성에 따른 고유 보안 문제

## 키워드
* 디폴트 패스워드, 펌웨어 취약점, Mirai 봇넷, 경량 암호화, 인증, OTA

## 암기법
* 디펌네물인: (취약점) 디폴트 패스워드, 펌웨어 취약점, 네트워크 취약점, 물리적 접근, 인증 미흡

## 특징
- 제한된 리소스: CPU, 메모리 제약으로 보안 기능 제한
- 대규모 연결: 수많은 디바이스로 공격 표면 확대
- 이기종 환경: 다양한 프로토콜, 표준 부재
- 물리적 노출: 디바이스 물리적 접근 용이

## 목적
- IoT 환경의 보안 위협 식별
- 효과적인 보안 대책 수립
- 안전한 IoT 서비스 운영

## 구성요소
| 구분 | 취약점 | 대응방안 |
|------|--------|----------|
| 디바이스 | 디폴트 패스워드 | 초기 비밀번호 변경 강제 |
| 디바이스 | 펌웨어 취약점 | 보안 업데이트, 서명 검증 |
| 디바이스 | 물리적 접근 | 하드웨어 보안, 탬퍼링 방지 |
| 네트워크 | 평문 통신 | 경량 암호화 (DTLS, CoAP) |
| 네트워크 | 프로토콜 취약점 | 보안 프로토콜 적용 |
| 인증 | 인증 미흡 | 디바이스 인증, PKI |
| 데이터 | 데이터 유출 | 암호화, 접근제어 |
| 관리 | 업데이트 미흡 | OTA(Over-The-Air) 업데이트 |

## 구성도
```
[IoT 보안 위협 및 대응]

[IoT 계층별 위협]
         │
    ┌────┴────┬────────────┐
    ▼         ▼            ▼
[디바이스]  [네트워크]    [플랫폼/클라우드]
    │         │            │
    ▼         ▼            ▼
[펌웨어]   [스니핑]      [API 취약점]
[물리접근] [DDoS]        [데이터 유출]
[디폴트PW] [MITM]        [인증 우회]

[대응 기술]
    │
    ├── 경량 암호화 (AES-128, ECC)
    ├── 디바이스 인증 (인증서, TPM)
    ├── 보안 부팅 (Secure Boot)
    ├── OTA 보안 업데이트
    └── 네트워크 세그멘테이션

[Mirai 봇넷 공격 사례]
[IoT 디바이스] ──▶ [디폴트 PW 취약점]
                        │
                  [Mirai 감염]
                        │
                  [DDoS 공격]
```
