#정보관리기술사 #보안 #인증 #FIDO #생체인증 #정리 #보안/인증/접근제어

## 정의
비밀번호 없는 생체 인증 표준, FIDO 1.0
- Fast IDentity Online의 약자로, 비밀번호 없이 생체인식 등을 활용한 간편하고 안전한 온라인 인증을 위한 기술 표준
- FIDO Alliance에서 제정한 UAF(Universal Authentication Framework)와 U2F(Universal 2nd Factor) 프로토콜 포함

## 키워드
* UAF, U2F, 생체인증, 공개키, 개인키, 인증자(Authenticator), FIDO Alliance

## 암기법
* UAF U2F: (프로토콜) UAF(비밀번호 없는 인증), U2F(2차 인증)

## 특징
- 비밀번호 대체: 생체인식으로 비밀번호 불필요 (UAF)
- 2차 인증: 기존 인증에 추가 보안 (U2F)
- 공개키 기반: 개인키는 디바이스에만 저장
- 프라이버시 보호: 생체정보 서버 전송 없음

## 목적
- 비밀번호 의존성 탈피
- 피싱, 중간자 공격 방지
- 사용자 편의성과 보안성 동시 확보

## 구성요소
| 구분    | 구성요소               | 설명               |
| ----- | ------------------ | ---------------- |
| 프로토콜  | UAF                | 비밀번호 없는 인증, 생체인식 |
| 프로토콜  | U2F                | 2차 인증, 보안키 활용    |
| 클라이언트 | FIDO Client        | 인증 요청 처리         |
| 인증자   | Authenticator      | 생체인식/보안키 장치      |
| 서버    | FIDO Server        | 공개키 등록/검증        |
| 키     | 개인키                | 디바이스 내 안전 저장     |
| 키     | 공개키                | 서버에 등록           |
| 프로세스  | 등록(Registration)   | 공개키 서버 등록        |
| 프로세스  | 인증(Authentication) | 개인키로 서명, 공개키로 검증 |
