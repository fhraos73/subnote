#정보관리기술사 #보안 #제로트러스트 #접근제어 #정리

## 정의
무신뢰 기반 보안 모델, 제로 트러스트(Zero Trust)
- "절대 신뢰하지 말고, 항상 검증하라(Never Trust, Always Verify)"는 원칙에 기반한 보안 모델
- 네트워크 내부/외부 구분 없이 모든 접근 요청에 대해 지속적인 인증과 검증을 수행하는 보안 아키텍처

## 키워드
* Never Trust Always Verify, 마이크로세그멘테이션, 최소권한, MFA, SASE, SDP

## 암기법
* 네항최마: (원칙) Never Trust, 항상 검증, 최소 권한, 마이크로세그멘테이션
* 신장사데: (구성요소) 신원, 장치, 네트워크/환경, 데이터

## 특징
- 경계 무의미: 내부/외부 네트워크 경계 구분 없음
- 지속적 검증: 모든 접근에 대한 실시간 인증
- 최소 권한: 필요한 최소한의 권한만 부여
- 마이크로세그멘테이션: 세분화된 접근 제어

## 목적
- 내부 위협 및 측면 이동(Lateral Movement) 방지
- 클라우드/원격 근무 환경 보안 강화
- 데이터 중심 보안 체계 구축

## 구성요소
| 구분 | 구성요소 | 설명 |
|------|----------|------|
| 원칙 | Never Trust | 모든 접근을 잠재적 위협으로 간주 |
| 원칙 | Always Verify | 지속적인 인증 및 검증 |
| 원칙 | 최소 권한 | 업무에 필요한 최소 권한만 부여 |
| 기술 | MFA | 다중 인증 (Multi-Factor Authentication) |
| 기술 | 마이크로세그멘테이션 | 네트워크 세분화 |
| 기술 | SDP | 소프트웨어 정의 경계 |
| 기술 | SASE | 보안 접근 서비스 엣지 |
| 기술 | IAM | 신원 및 접근 관리 |
| 검증 대상 | 신원(Identity) | 사용자/서비스 신원 검증 |
| 검증 대상 | 장치(Device) | 디바이스 상태 검증 |
| 검증 대상 | 환경(Context) | 접근 환경/위치 검증 |
