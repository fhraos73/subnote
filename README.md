## 핵심 파일 역할

### `0. Keyword Note.md`

전체 토픽을 한눈에 조회할 수 있는 **마스터 테이블**입니다.

- 모든 토픽이 분류별로 묶인 **H3 섹션**으로 구분됩니다.
- 각 섹션 아래에 테이블 형식으로 토픽이 정리됩니다.
- 컬럼 구성: `No | 도메인 | 토픽 | 정의 | 암기법`
- 토픽 이름은 Obsidian 내부 링크(`[[...]]`)로 연결되어 있어 바로 서브노트로 이동 가능합니다.
- **용도**: 전체 토픽 빠르게 훑어보기, 암기법 일괄 확인, 학습 범위 파악

### `00. key_list.md`

분류별 **태그와 토픽 링크 목록**을 정리한 인덱스 파일입니다.

- `0. Keyword Note.md`의 각 섹션에 대응하는 태그(`#IT경영전략/경영전략/분석_기법` 등)가 헤더 아래에 기록됩니다.
- 해당 분류에 속하는 모든 토픽이 `[[링크]]` 형태로 나열됩니다.
- **용도**: 분류별 토픽 목록 확인, Obsidian 태그 검색 연동, 토픽 누락 여부 확인

```md
### IT경영전략 - 경영전략/분석 기법
#IT경영전략/경영전략/분석_기법

- [[3C, 4C]]
- [[SWOT분석]]
- [[MECE, LISS]]
...
```

- 특징2: 간단명료 설명

## 태그 체계

서브노트 상단에 아래 형식으로 태그가 작성됩니다.

```
#도메인 #세부주제 #분류태그
```

| 태그 유형 | 예시 | 설명 |
|---|---|---|
| 도메인 | `#IT경영전략`, `#SW공학`, `#AI` | 대분류 도메인 |
| 세부주제 | `#아키텍처`, `#암호화`, `#애자일` | 내용 기반 분류 |
| 중요도 | `#필수`, `#암기` | 시험 대비 중요도 |
| 분류 | `#IT경영전략/경영전략/분석_기법` | `key_list.md` 연동용 계층 태그 |

## 파일 간 연관 관계

```
key_list.md
  └─ 분류별 태그 + 토픽 [[링크]] 목록
        │
        ▼
0. Keyword Note.md
  └─ 전체 토픽 테이블 (No / 도메인 / 토픽 / 정의 / 암기법)
        │  [[링크]]
        ▼
도메인별 서브노트 (.md)
  └─ 상세 내용 (정의 / 키워드 / 암기법 / 특징 / 목적 / 구성요소 / 구성도 / 연관 토픽)
```
