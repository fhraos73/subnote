	- RAID 5, A, B, C , Parity(복구용, XOR 연산 복구), 2개 이상 고장 시 복구 x
	- RAID 6, 2개 패리티
	- RAID 10, 1 + 0

14). (2교시형 확정) FP, EVA, AI 비용 산정
- FP : 
	- ILF, EIF, EQ, EI, EO
	- 605784원, 규모 보정계수 1.28
- EVA :
	- 지표 : BAC, PV(계획), EV(획득), AC(비용)
	- 성과 지수 : CV(비용편차), SV(일정편차), CPI(비용성과지수), SPI(일정성과지수)
		- CV = EV-AC
		- SV = EV-PV
		- CPI = EV/AC
		- SPI = EV/PV
	- 예측 지표 : EAC(현추세 완료 예상비용), ETC(앞으로 비용), VAC(완료 시 비용 편차 예측)
		- EAC = BAC/CPI
		- ETC = EAC-AC
		- VAC = BAC - EAC
12). 팬텀충돌과 동시성제어
- 팬텀충돌 동일한 조건 두번 읽기 → 데이터가 사라지거나 나타나는 현상
- 팬텀읽기로 인한 충돌
- Insert, Delete 집합 변화 이상현상
- → Gap lock, Next Key Lock, Snapshot
- 동시성 제어 격리성 보장의 딜레마 (동시성 <> 일관성 트레이드오프)
- Read Uncommited, Read Commited, Repeat Read, Serializeble
- 동시성 제어 기법
	- 비관적 동시성 제어 (Pessimistic CC) 
		- 2PL (2단계 잠금)
		- 타임스탬프 기반
		- 다중 세분도 잠금
	- 낙관적 동시성 제어 (Optimistic CC)
		- OCC (검증 후 커밋)
	- 다중 버전 동시성 제어
		- MVCC
- 2PL, OCC(낙관적 읽기 많은 환경 유리), MVCC(버전 기반, 읽기-쓰기 비차단, 현재 DB 표준)
