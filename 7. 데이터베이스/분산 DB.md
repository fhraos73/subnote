#데이터베이스 #분산시스템 #확장성 #필수
## 정의
지리적으로 분산된 데이터베이스, 분산 DB
- 네트워크로 연결된 여러 사이트에 데이터를 분산 저장하고 통합 관리하는 시스템
- 위치 투명성, 복제 투명성, 분할 투명성을 제공하여 단일 DB처럼 사용
## 키워드
* 투명성, 2PC, 복제, 분할, CAP, 분산 트랜잭션
## 암기법
* "위복분장병" - 위치, 복제, 분할, 장애, 병행 투명성
## 분산 DB 투명성
| 투명성 | 설명 |
|--------|------|
| 위치 투명성 | 데이터 물리적 위치 은폐 |
| 복제 투명성 | 데이터 복제본 존재 은폐 |
| 분할 투명성 | 데이터 분할 방식 은폐 |
| 장애 투명성 | 장애 발생 시에도 서비스 |
| 병행 투명성 | 동시 접근 제어 은폐 |
## 구성도
```
           ┌─────────────┐
           │ 분산 DBMS   │
           │ (조정자)    │
           └──────┬──────┘
      ┌───────────┼───────────┐
      ▼           ▼           ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│ Site A  │ │ Site B  │ │ Site C  │
│ (서울)  │ │ (부산)  │ │ (대전)  │
└─────────┘ └─────────┘ └─────────┘
```
## 2PC (2단계 커밋)
```
[조정자]                [참여자들]
   │                       │
   │──── Prepare ─────────→│
   │←─── Ready/Abort ──────│
   │                       │
   │──── Commit/Rollback ─→│
   │←─── ACK ──────────────│
```
## 분산 DB 장단점
| 장점 | 단점 |
|------|------|
| 확장성 | 복잡한 관리 |
| 가용성 | 네트워크 의존 |

| 성능 | 분산 트랜잭션 오버헤드 |
