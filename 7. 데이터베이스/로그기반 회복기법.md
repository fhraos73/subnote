#데이터베이스 #회복기법 #로그 #Undo #Redo #필수 #데이터베이스/트랜잭션/회복
## 정의
로그 기반 장애 복구, Log-Based Recovery
- 트랜잭션 수행 중 변경된 데이터를 로그에 기록하여 장애 발생 시 복구하는 기법
- Write-Ahead Logging(WAL) 원칙에 따라 데이터 변경 전 로그를 먼저 기록

## 키워드
* WAL (Write-Ahead Logging): 로그 선기록 원칙
* Undo Log: 이전 값(Before Image) 기록, Rollback용
* Redo Log: 이후 값(After Image) 기록, 재실행용
* 로그 레코드: <Ti, X, V1, V2> (트랜잭션ID, 데이터항목, 이전값, 이후값)
* LSN (Log Sequence Number): 로그 레코드 식별자

## 암기법
* 회복 방향: "Undo는 뒤로(Backward), Redo는 앞으로(Forward)"
* WAL: "왈! 로그 먼저" (Write-Ahead Logging)
* 로그 내용: "티엑스 비원비투" (<Ti, X, V1, V2>)

## 구성요소/특징/유형
| 구분 | Undo 로그 | Redo 로그 | Undo/Redo 로그 |
|------|-----------|-----------|----------------|
| 기록 내용 | Before Image (V1) | After Image (V2) | Both (V1, V2) |
| 기록 시점 | 데이터 변경 전 | Commit 전 | 데이터 변경 전 |
| 디스크 반영 | Commit 전 강제 출력 | Commit 후 가능 | 유연함 |
| 복구 방향 | Backward (역방향) | Forward (정방향) | 양방향 |
| 장애 복구 | 미완료 트랜잭션 취소 | 완료 트랜잭션 재실행 | 취소 + 재실행 |

### 회복 연산
| 연산 | 대상 | 동작 | 사용 시점 |
|------|------|------|-----------|
| Undo | 미완료 트랜잭션 | Before Image로 복원 | Commit 없는 트랜잭션 |
| Redo | 완료 트랜잭션 | After Image로 재실행 | Commit 있으나 미반영 |

### 로그 레코드 유형
| 레코드 | 형식 | 설명 |
|--------|------|------|
| Start | <Ti, Start> | 트랜잭션 시작 |
| Update | <Ti, X, V1, V2> | 데이터 변경 |
| Commit | <Ti, Commit> | 트랜잭션 완료 |
| Abort | <Ti, Abort> | 트랜잭션 취소 |
