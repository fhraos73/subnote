#데이터베이스 #옵티마이저 #실행계획 #필수
## 정의
SQL 최적 실행계획 수립기
- SQL문을 분석하여 가장 효율적인 실행계획을 수립하는 DBMS 핵심 엔진
- 규칙기반(RBO)과 비용기반(CBO) 두 가지 방식으로 분류
## 키워드
* RBO(Rule-Based Optimizer), CBO(Cost-Based Optimizer)
* 실행계획(Execution Plan), 통계정보(Statistics)
* 카디널리티, 선택도, 비용(Cost), 힌트(Hint)
## 암기법
* "RBO는 규칙, CBO는 비용" - 규칙기반 vs 비용기반
* "통카선비" - 통계정보, 카디널리티, 선택도, 비용
## 구성요소/특징/유형
### RBO vs CBO 비교
| 구분 | RBO (규칙기반) | CBO (비용기반) |
|------|---------------|---------------|
| 최적화 기준 | 고정된 우선순위 규칙 | 비용(I/O, CPU, 메모리) |
| 통계정보 | 사용 안함 | 적극 활용 |
| 유연성 | 낮음 | 높음 |
| 실행계획 | 규칙 우선순위 기반 | 최소 비용 경로 선택 |
| 현재 상태 | Oracle 10g 이후 폐기 | 현대 DBMS 표준 |
### RBO 우선순위 규칙 (15단계)
- 1순위: Single row by ROWID (가장 빠름)
- 4순위: Single row by Unique Index
- 9순위: Index Range Scan
- 15순위: Full Table Scan (가장 느림)
### CBO 비용 결정 인자
- **레코드 블록 수**: 테이블 크기
- **평균 행 길이**: 데이터 밀도
- **인덱스 높이**: B-Tree 깊이
- **컬럼 값 분포**: 히스토그램
- **디스크 I/O**: 물리적 접근 비용
### CBO 동작 과정
1. SQL문 파싱 및 분석
2. 통계정보 수집/참조
3. 후보 실행계획 생성
4. 각 계획별 비용 계산
5. 최소 비용 계획 선택
