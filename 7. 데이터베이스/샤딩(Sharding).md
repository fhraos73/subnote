#데이터베이스 #분산DB #확장성 #필수
## 정의
대용량 데이터 분산 저장, 샤딩
- 대규모 데이터를 여러 데이터베이스 서버에 수평 분할하여 저장하는 기법
- 각 샤드(Shard)는 독립적인 데이터베이스로 동작하며 확장성 확보
## 키워드
* Shard, Shard Key, 수평 분할, Hash Sharding, Range Sharding
## 암기법
* "해레디" - Hash, Range, Directory 기반 샤딩
## 샤딩 전략
| 전략 | 설명 | 장단점 |
|------|------|--------|
| Hash Sharding | 해시 함수로 샤드 결정 | 균등 분배, 범위 검색 어려움 |
| Range Sharding | 키 범위로 샤드 결정 | 범위 검색 용이, 핫스팟 발생 |
| Directory Sharding | 매핑 테이블로 샤드 결정 | 유연함, 조회 오버헤드 |
| Geo Sharding | 지역 기반 샤드 결정 | 지역성 활용, 관리 복잡 |
## 구성도
```
           ┌─────────────┐
           │  Application │
           └──────┬──────┘
                  │
           ┌──────▼──────┐
           │ Shard Router │
           └──────┬──────┘
      ┌───────────┼───────────┐
      ▼           ▼           ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│ Shard 1 │ │ Shard 2 │ │ Shard 3 │
│ (A~G)   │ │ (H~N)   │ │ (O~Z)   │
└─────────┘ └─────────┘ └─────────┘
```
## 샤딩 vs 파티셔닝
| 구분 | 샤딩 | 파티셔닝 |
|------|------|----------|
| 범위 | 여러 서버 | 단일 서버 |
| 목적 | 확장성 | 성능/관리 |
| 복잡도 | 높음 | 낮음 |
