#데이터베이스 #이상현상 #정규화 #필수
## 정의
데이터 중복으로 인한 이상현상
- 릴레이션 조작 시 데이터 중복으로 발생하는 삽입, 삭제, 갱신 시의 비합리적 현상
- 정규화되지 않은 테이블에서 여러 사실을 하나의 릴레이션으로 표현할 때 발생
## 키워드
* 삽입이상(Insertion Anomaly): 불필요한 정보 함께 삽입
* 삭제이상(Deletion Anomaly): 유용한 정보 함께 삭제
* 갱신이상(Update Anomaly): 일부만 수정 시 불일치
* 데이터 중복, 정규화, 함수적 종속성
## 암기법
* "삽삭갱" - 삽입/삭제/갱신 이상
* "중이정" - 중복이 이상을 만들고, 정규화로 해결
## 구성요소/특징/유형
### 이상현상의 3가지 유형
| 유형 | 설명 | 예시 |
|------|------|------|
| 삽입이상 | 새 데이터 삽입 시 불필요한 데이터도 함께 삽입 | 신규 부서 등록 시 소속 사원 없으면 NULL 입력 필요 |
| 삭제이상 | 튜플 삭제 시 유용한 정보도 함께 삭제 | 마지막 사원 퇴사 시 부서 정보도 삭제 |
| 갱신이상 | 중복 튜플 중 일부만 수정 시 데이터 불일치 | 부서 전화번호 변경 시 일부 행만 수정 |
### 발생 원인
- 데이터 중복 저장
- 함수적 종속성 미고려
- 부적절한 테이블 설계
### 해결 방안
- 정규화(Normalization) 수행
- 테이블 분리를 통한 중복 제거
- 함수적 종속성 분석
## 구성도

```
│                    비정규화 테이블                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 사원번호 │ 사원명 │ 부서코드 │ 부서명 │ 부서전화 │      │
│  ├─────────────────────────────────────────────────────┤   │
│  │   001   │ 홍길동 │   D01   │ 개발부 │ 1234    │      │
│  │   002   │ 김철수 │   D01   │ 개발부 │ 1234    │ ← 중복│
│  │   003   │ 이영희 │   D02   │ 기획부 │ 5678    │      │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌──────────────────┐  ┌──────────────────────────────┐    │
│  │ 삽입이상: 부서만   │  │ 갱신이상: D01 전화번호 변경 시 │    │
│  │ 추가 불가         │  │ 일부만 수정되면 불일치        │    │
│  └──────────────────┘  └──────────────────────────────┘    │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ 삭제이상: 003 삭제 시 기획부 정보도 함께 삭제          │  │
│  └──────────────────────────────────────────────────────┘  │
│                          ↓ 정규화                           │
│  ┌─────────────────┐    ┌────────────────────────┐        │
│  │ 사원 테이블      │    │ 부서 테이블             │        │
│  │ 사원번호│사원명  │    │ 부서코드│부서명│부서전화│        │
│  │  001  │홍길동   │    │  D01  │개발부│ 1234  │        │
│  │  002  │김철수   │    │  D02  │기획부│ 5678  │        │
│  └─────────────────┘    └────────────────────────┘        │
└─────────────────────────────────────────────────────────────┘
```
