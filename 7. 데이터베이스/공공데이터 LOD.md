#데이터베이스 #공공데이터 #LOD #시맨틱웹 #필수 #데이터베이스/빅데이터/분석
## 정의
연결된 개방형 데이터
- 웹에서 데이터를 URI로 식별하고 RDF 형식으로 연결하여 공개하는 데이터 개방 방식
- 공공데이터를 기계가 이해 가능한 형태로 연결하여 활용성을 극대화
## 키워드
* Linked Open Data, URI, RDF, SPARQL
* 트리플(Triple), 온톨로지(Ontology), 시맨틱웹
* 5-Star Open Data, 데이터 연계
## 암기법
* "LOD는 유알스온" - URI, RDF, SPARQL, Ontology
* "주술목트리플" - 주어(Subject), 술어(Predicate), 목적어(Object) = Triple
## 구성요소/특징/유형
### LOD 4대 원칙 (Tim Berners-Lee)
| 원칙 | 설명 |
|------|------|
| URI 사용 | 모든 객체에 고유 URI 부여 |
| HTTP URI | HTTP 프로토콜로 접근 가능한 URI |
| RDF 형식 | 표준화된 데이터 표현 (트리플) |
| 링크 연결 | 다른 데이터와 연결 정보 포함 |
### 5-Star Open Data 등급
| 등급 | 형식 | 설명 |
|------|------|------|
| ★ | PDF, 이미지 | 개방 라이선스로 공개 |
| ★★ | Excel | 구조화된 데이터 |
| ★★★ | CSV, XML | 비독점 형식 |
| ★★★★ | RDF | URI로 식별 가능 |
| ★★★★★ | LOD | 다른 데이터와 연결 |
### LOD 핵심 기술
| 기술 | 설명 |
|------|------|
| URI | 자원의 고유 식별자 |
| RDF | 트리플(주어-술어-목적어) 데이터 모델 |
| SPARQL | RDF 데이터 질의 언어 |
| OWL | 온톨로지 정의 언어 |
| RDFS | RDF 스키마 정의 |
### 국내 LOD 서비스 현황
| 기관 | 서비스 | 내용 |
|------|--------|------|
| 국립중앙도서관 | 국가서지 LOD | 도서, 저자, 주제명 정보 |
| 서울시 | 열린데이터광장 LOD | 행정, 문화, 교통 데이터 |
| 공공데이터포털 | 연계데이터 | 메타데이터 LOD |
| 국사편찬위원회 | 한국사 LOD | 역사 데이터 |
### LOD 장점
- **중복 방지**: 동일 데이터 재생성 불필요
- **연계 용이**: 웹 전체를 데이터베이스로 활용
- **표준화**: RDF 표준으로 상호운용성 확보
- **확장성**: 새로운 데이터 연결 용이
- **기계 가독성**: AI/자동화 처리 가능
### LOD 과제
- 공공데이터 LOD 변환 비용
- 데이터 품질 관리 체계 필요
- 활용 사례 및 인식 부족
- 민간 데이터와의 연계 확대
