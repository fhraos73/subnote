#데이터베이스 #설계 #용량 #필수 #데이터베이스/성능/튜닝
## 정의
데이터베이스 저장 공간 예측, 용량산정
- 데이터베이스 구축 시 필요한 저장 공간을 사전에 예측하고 산정하는 활동
- 테이블, 인덱스, 로그, 백업 등 전체 용량을 고려하여 인프라 규모 결정
## 키워드
* 행 길이, 행 수, 증가율, 인덱스 용량, 여유율
## 암기법
* "테인로백여" - 테이블, 인덱스, 로그, 백업, 여유
## 용량산정 공식
```
테이블 용량 = 행 길이 × 행 수 × (1 + 증가율)
인덱스 용량 = 테이블 용량 × 0.3~0.5 (경험치)
전체 용량 = (테이블 + 인덱스 + 로그 + 백업) × 여유율
```
## 산정 절차
```
[요건분석] → [데이터모델] → [용량계산] → [인프라결정]
 업무분석    테이블설계    공식적용    서버/스토리지
 증가율예측  인덱스설계    여유율적용   용량확정
```
## 산정 항목
| 항목 | 산정 기준 | 비고 |
|------|-----------|------|
| 테이블 | 행길이 × 행수 | 데이터타입별 계산 |
| 인덱스 | 키길이 × 행수 × 1.3 | B-Tree 오버헤드 |
| Redo 로그 | 트랜잭션량 기반 | 피크 시간 고려 |
| 백업 | 전체 용량 × 보관주기 | 압축률 적용 |
| 여유율 | 전체 × 1.3~1.5 | 성장률 반영 |
## 구성도
```
┌─────────────────────────────────────────┐
│           전체 DB 용량                   │
├──────────┬──────────┬──────────┬────────┤
│ 테이블   │ 인덱스   │ 로그     │ 백업   │
│ (60%)    │ (20%)    │ (10%)    │ (10%) │
├──────────┴──────────┴──────────┴────────┤
│              + 여유율 (30~50%)           │
└─────────────────────────────────────────┘
```
