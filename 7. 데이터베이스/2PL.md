#데이터베이스 #동시성제어 #2PL #잠금 #필수 #데이터베이스/동시성제어
## 정의
2단계 잠금 프로토콜, 2PL
- Two-Phase Locking Protocol
- 트랜잭션의 모든 Lock 연산이 첫 번째 Unlock 연산보다 선행하도록 보장하는 프로토콜
- 직렬 가능성(Serializability)을 보장하는 동시성 제어 기법

## 키워드
* 2단계: Growing Phase (확장), Shrinking Phase (축소)
* Lock Point: Growing → Shrinking 전환 시점
* Lock 유형: Shared Lock (S-Lock), Exclusive Lock (X-Lock)
* 변형: Basic 2PL, Strict 2PL, Rigorous 2PL

## 암기법
* 2PL: "확축" (확장 단계 → 축소 단계)
* 규칙: "잠금 다 걸고 해제 시작" (Growing 완료 후 Shrinking)
* Lock 유형: "쉐엑" (Shared, Exclusive)
* 변형: "베스릭" (Basic, Strict, Rigorous)

## 구성요소/특징/유형
### 2PL 단계
| 단계 | 영문 | 동작 | 제약 |
|------|------|------|------|
| 확장 단계 | Growing Phase | Lock 획득만 가능 | Unlock 불가 |
| 축소 단계 | Shrinking Phase | Unlock만 가능 | 새 Lock 불가 |

### Lock 유형
| Lock | 영문 | 용도 | 호환성 |
|------|------|------|--------|
| S-Lock | Shared Lock | 읽기 연산 | S-Lock과 호환 |
| X-Lock | Exclusive Lock | 쓰기 연산 | 모든 Lock과 비호환 |

### Lock 호환성 매트릭스
|  | S-Lock | X-Lock |
|--|--------|--------|
| S-Lock | O (호환) | X (비호환) |
| X-Lock | X (비호환) | X (비호환) |

### 2PL 변형
| 변형 | 설명 | 장점 | 단점 |
|------|------|------|------|
| Basic 2PL | 기본 2단계 잠금 | 직렬 가능성 보장 | 연쇄 복귀 가능 |
| Strict 2PL | X-Lock은 Commit까지 유지 | 연쇄 복귀 방지 | 동시성 저하 |
| Rigorous 2PL | 모든 Lock Commit까지 유지 | 완전한 격리 | 동시성 최저 |

### 2PL의 한계
| 문제 | 설명 |
|------|------|
| Deadlock | 두 트랜잭션이 서로의 Lock 대기 |
| 연쇄 복귀 | Basic 2PL에서 발생 가능 |
| 동시성 저하 | Lock 대기로 인한 성능 저하 |
