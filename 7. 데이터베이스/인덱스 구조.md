#데이터베이스 #인덱스 #B-Tree #Hash #Bitmap #필수 #데이터베이스/성능/튜닝
## 정의
데이터 검색 최적화 구조
- 테이블의 데이터를 빠르게 검색하기 위한 별도의 자료구조
- 구조에 따라 B-Tree, Hash, Bitmap 등으로 분류되며 용도별 선택 필요
## 키워드
* B-Tree 인덱스, B+Tree, Hash 인덱스, Bitmap 인덱스
* 루트/브랜치/리프 노드, 해시 함수, 비트맵 연산
* OLTP, OLAP, 카디널리티, 선택도
## 암기법
* "B는 범위, H는 해시, Bit는 비트" - 범위검색(B-Tree), 동등검색(Hash), 저카디널리티(Bitmap)
* "OLTP는 B, OLAP는 Bit" - 트랜잭션은 B-Tree, 분석은 Bitmap
## 구성요소/특징/유형
### 인덱스 구조별 비교
| 구분 | B-Tree | Hash | Bitmap |
|------|--------|------|--------|
| 검색 방식 | 범위/동등 검색 | 동등 검색만 | 비트 연산 |
| 시간복잡도 | O(log n) | O(1) | O(n) |
| 적합 환경 | OLTP | Key-Value | OLAP |
| 카디널리티 | 높음 | 높음 | 낮음 |
| 범위 검색 | 우수 | 불가 | 가능 |
| 갱신 성능 | 양호 | 양호 | 불량 |
### B-Tree 인덱스
- **구조**: 루트 → 브랜치 → 리프 노드 (균형 트리)
- **탐색**: 수직적 탐색(시작점) + 수평적 탐색(범위)
- **장점**: 범위 검색, ORDER BY, GROUP BY 최적화
- **단점**: 삽입/수정/삭제 시 재균형 비용
- **적용**: 대부분의 OLTP 환경
### Hash 인덱스
- **구조**: 해시 함수 → 해시 버킷 → 데이터 위치
- **탐색**: Key → Hash Value → Direct Access
- **장점**: 동등 비교(=) 시 O(1) 접근
- **단점**: 범위 검색(<, >, BETWEEN) 불가, 정렬 불가
- **적용**: 메모리 DB, Key-Value 저장소
### Bitmap 인덱스
- **구조**: 각 값별 비트맵 (0/1 배열)
- **탐색**: 비트맵 간 AND/OR 연산
- **장점**: 저카디널리티 컬럼에 효율적 (성별, 상태값)
- **단점**: 갱신 시 모든 비트맵 수정 필요
- **적용**: DW, OLAP 환경의 다중 조건 분석
## 구성도
```
┌─────────────────────────────────────────────────────┐
│              인덱스 구조별 비교                       │
├─────────────────────────────────────────────────────┤
│  [B-Tree 인덱스]                                    │
│           ┌───┐                                     │
│           │ R │ (Root)                              │
│           └─┬─┘                                     │
│        ┌───┴───┐                                    │
│      ┌─┴─┐   ┌─┴─┐ (Branch)                        │
│      │ B │   │ B │                                  │
│      └─┬─┘   └─┬─┘                                  │
│    ┌──┴──┐ ┌──┴──┐                                 │
│   ┌┴┐  ┌┴┐┌┴┐  ┌┴┐ (Leaf → 데이터 포인터)          │
│   │L│──│L│L│──│L│  (수평 연결)                     │
│   └─┘  └─┘└─┘  └─┘                                 │
├─────────────────────────────────────────────────────┤
│  [Hash 인덱스]                                      │
│   Key ──▶ Hash Function ──▶ Bucket ──▶ Data        │
│   "ABC"      h("ABC")=3       [3]      Row Pointer │
├─────────────────────────────────────────────────────┤
│  [Bitmap 인덱스]                                    │
│   성별 컬럼 (M/F)                                   │
│   Row:  1  2  3  4  5  6                           │
│   M:    1  0  1  0  1  0                           │
│   F:    0  1  0  1  0  1                           │
│   → M AND 서울 = 비트맵 AND 연산                    │
└─────────────────────────────────────────────────────┘
```
