#데이터베이스 #정규화 #BCNF #필수 #데이터베이스/정규화
## 정의
결정자가 후보키, BCNF
- 릴레이션의 모든 결정자(Determinant)가 후보키(Candidate Key)인 정규화 단계
- 3NF를 만족하면서 후보키가 아닌 결정자를 제거하여 이상현상을 완전히 해결
## 키워드
* 결정자(Determinant): 다른 속성의 값을 결정하는 속성
* 후보키(Candidate Key): 튜플을 유일하게 식별하는 최소 속성 집합
* Boyce-Codd Normal Form: 3NF의 강화된 형태
* 무손실 분해: 정보 손실 없는 테이블 분리
## 암기법
* "결후" - 결정자는 반드시 후보키
* "비결제" - 비후보키 결정자 제거
* "삼점오" - 3.5NF = BCNF
## 구성요소/특징/유형
### BCNF 조건
| 조건 | 설명 |
|------|------|
| 3NF 만족 | 이행함수종속 없음 |
| 모든 결정자가 후보키 | 비후보키가 결정자가 되면 안 됨 |
### 3NF vs BCNF 차이
| 구분 | 3NF | BCNF |
|------|-----|------|
| 조건 | 비키→비키 종속 제거 | 모든 결정자가 후보키 |
| 강도 | 약함 | 강함 |
| 종속성 보존 | 보장 | 보장 안 될 수 있음 |
| 무손실 분해 | 보장 | 보장 |
### BCNF 위반 조건
- 후보키가 아닌 속성이 다른 속성을 결정
- 후보키의 일부가 다른 속성에 의해 결정
### BCNF 변환 시 고려사항
| 항목 | 설명 |
|------|------|
| 종속성 보존 | BCNF 분해 시 일부 FD 손실 가능 |
| 실무 적용 | 종속성 보존이 중요하면 3NF에서 멈춤 |
| 트레이드오프 | 무결성 vs 종속성 보존 |
