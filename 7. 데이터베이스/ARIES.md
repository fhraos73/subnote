#데이터베이스 #회복기법 #ARIES #필수 #데이터베이스/트랜잭션/회복
## 정의
3단계 회복 알고리즘, ARIES
- Algorithm for Recovery and Isolation Exploiting Semantics
- IBM에서 개발한 WAL 기반의 3단계(분석-Redo-Undo) 회복 알고리즘
- 대부분의 상용 DBMS에서 채택한 표준 회복 기법

## 키워드
* 3단계: Analysis → Redo → Undo
* WAL (Write-Ahead Logging): 로그 선기록 원칙
* LSN (Log Sequence Number): 로그 레코드 고유 식별자
* Fuzzy Checkpoint: 트랜잭션 중단 없는 체크포인트
* CLR (Compensation Log Record): Undo 작업의 로그 기록

## 암기법
* ARIES: "아리스는 분리언" (분석, Redo, Undo 3단계)
* 3단계 순서: "분리언" (분석 → Redo → Undo)
* 핵심 원칙: "왈퍼리클" (WAL, Fuzzy, Repeating History, CLR)

## 구성요소/특징/유형
### ARIES 3단계
| 단계 | 영문 | 동작 | 목적 |
|------|------|------|------|
| 1단계 | Analysis | 체크포인트부터 로그 분석 | Redo/Undo 대상 결정 |
| 2단계 | Redo | 완료+미완료 모든 트랜잭션 재실행 | 장애 직전 상태 복원 |
| 3단계 | Undo | 미완료 트랜잭션 취소 | 일관성 있는 상태 복원 |

### ARIES 핵심 원칙
| 원칙 | 설명 |
|------|------|
| WAL | 데이터 변경 전 로그 먼저 기록 |
| Repeating History | Redo 시 완료/미완료 구분 없이 모두 재실행 |
| Logging During Undo | Undo 작업도 CLR로 로그 기록 |
| Fuzzy Checkpoint | 체크포인트 중 트랜잭션 수행 허용 |

### 주요 자료구조
| 자료구조 | 설명 | 용도 |
|----------|------|------|
| Log | 모든 변경 기록 | 회복의 기본 데이터 |
| Transaction Table | 활성 트랜잭션 목록 | Undo 대상 결정 |
| Dirty Page Table | 변경된 페이지 목록 | Redo 시작점 결정 |

### Analysis 단계 상세
- 체크포인트 레코드에서 Transaction Table, Dirty Page Table 로드
- 로그를 순방향 스캔하며 테이블 갱신
- Redo 시작 LSN (RedoLSN) 결정
- Undo 대상 트랜잭션 목록 결정
