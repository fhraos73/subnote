#데이터베이스 #무결성 #제약조건 #필수 #데이터베이스/트랜잭션/회복
## 정의
데이터 정확성 보장, 무결성 제약조건
- 데이터베이스에 저장된 데이터의 정확성, 일관성, 유효성을 보장하기 위한 규칙
- 부적절한 데이터 입력을 방지하여 데이터베이스의 신뢰성을 확보하는 메커니즘

## 키워드
* 3대 무결성: 개체 무결성, 참조 무결성, 속성 무결성, 사용자정의 무결성, 키 무결성, 도메인 무결성
* 제약조건: PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL, CHECK, DEFAULT
* 무결성 규칙: 선언적 무결성, 절차적 무결성
* 참조 무결성 옵션: CASCADE, SET NULL, SET DEFAULT, NO ACTION, RESTRICT

## 암기법
* 무결성 유형: "개참속사키도" (개체, 참조, 속성, 사용자정의, 키, 도메인)
* 제약조건: "프포유낫체디" (PK, FK, Unique, Not Null, Check, Default)
* 참조 옵션: "캐셋셋노리" (Cascade, Set Null, Set Default, No Action, Restrict)

## 구성요소/특징/유형
| 무결성 유형 | 설명 | 제약조건 | 예시 |
|------------|------|----------|------|
| 개체 무결성 (Entity) | 기본키는 NULL 불가, 중복 불가 | PRIMARY KEY | 학번은 유일하고 NULL 불가 |
| 참조 무결성 (Referential) | 외래키는 참조 테이블의 PK 또는 NULL | FOREIGN KEY | 수강.학번 → 학생.학번 |
| 속성 무결성 (Attribute) | 속성의 기본값, NULL 허용 여부 정의 | NOT NULL, DEFAULT | 입학일 기본값 SYSDATE |
| 사용자 정의 무결성 (User-defined) | 업무 규칙에 따른 사용자 정의 제약 | CHECK, TRIGGER | 나이는 0~150 사이 |
| 키 무결성 (Key) | 릴레이션에 최소 하나의 키 존재 보장 | UNIQUE, PRIMARY KEY | 후보키는 유일성 보장 |
| 도메인 무결성 (Domain) | 속성값은 정의된 도메인 내 값만 허용 | CHECK, 데이터타입 | 학점은 0.0~4.5 |

### 참조 무결성 옵션
| 옵션 | 부모 삭제/수정 시 동작 |
|------|------------------------|
| CASCADE | 자식 레코드도 함께 삭제/수정 |
| SET NULL | 자식의 외래키를 NULL로 설정 |
| SET DEFAULT | 자식의 외래키를 기본값으로 설정 |
| NO ACTION | 참조하는 자식 있으면 거부 (지연 검사) |
| RESTRICT | 참조하는 자식 있으면 즉시 거부 |

### 무결성 보장 방법
| 방법 | 설명 | 예시 |
|------|------|------|
| 선언적 무결성 | DDL로 제약조건 선언 | CREATE TABLE ... CONSTRAINT |
| 절차적 무결성 | 트리거, 프로시저로 구현 | BEFORE INSERT TRIGGER |
