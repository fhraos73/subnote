## 정의
트랜잭션 격리 수준, Isolation Level
- 동시에 실행되는 트랜잭션들이 서로에게 영향을 미치는 정도를 제어하는 수준
- 격리 수준이 높을수록 데이터 정합성 보장, 동시성(성능) 저하의 Trade-off 관계

## 키워드
* 4단계: Read Uncommitted, Read Committed, Repeatable Read, Serializable
* 이상현상: Dirty Read, Non-Repeatable Read, Phantom Read
* ANSI/ISO SQL-92 표준 정의
* 격리수준 ↑ = 정합성 ↑, 동시성 ↓

## 암기법
* 격리수준: "언커리시" (Uncommitted, Committed, Repeatable, Serializable)
* 이상현상: "더논팬" (Dirty, Non-repeatable, Phantom)
* 수준별 허용: "더(0)→논(1)→팬(2)→없(3)"

## 구성요소/특징/유형
| 격리 수준 | Dirty Read | Non-Repeatable Read | Phantom Read | 설명 |
|-----------|------------|---------------------|--------------|------|
| Read Uncommitted (Level 0) | O | O | O | 커밋 안된 데이터 읽기 허용 |
| Read Committed (Level 1) | X | O | O | 커밋된 데이터만 읽기 |
| Repeatable Read (Level 2) | X | X | O | 트랜잭션 내 동일 쿼리 동일 결과 |
| Serializable (Level 3) | X | X | X | 완전한 격리, 순차 실행과 동일 |

### 이상현상 상세
| 이상현상 | 설명 | 예시 |
|----------|------|------|
| Dirty Read | 커밋되지 않은 데이터 읽기 | T1이 변경 중인 값을 T2가 읽음 |
| Non-Repeatable Read | 같은 쿼리 다른 결과 (UPDATE) | T1이 두 번 읽는 사이 T2가 수정 |
| Phantom Read | 같은 쿼리 다른 행 수 (INSERT/DELETE) | T1이 두 번 읽는 사이 T2가 삽입 |

## 구성도
```
┌─────────────────────────────────────────────────────────────────┐
│                    Isolation Level 계층 구조                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   격리수준 ↑        정합성 ↑        동시성 ↓        성능 ↓       │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ Level 3: Serializable                                   │   │
│   │ - 완전한 격리 보장                                       │   │
│   │ - Phantom Read 방지 (범위 잠금)                          │   │
│   ├─────────────────────────────────────────────────────────┤   │
│   │ Level 2: Repeatable Read                                │   │
│   │ - 읽은 데이터 재읽기 보장                                │   │
│   │ - Non-Repeatable Read 방지                              │   │
│   ├─────────────────────────────────────────────────────────┤   │
│   │ Level 1: Read Committed (대부분 DBMS 기본값)            │   │
│   │ - 커밋된 데이터만 읽기                                   │   │
│   │ - Dirty Read 방지                                       │   │
│   ├─────────────────────────────────────────────────────────┤   │
│   │ Level 0: Read Uncommitted                               │   │
│   │ - 커밋 안된 데이터도 읽기                                │   │
│   │ - 모든 이상현상 발생 가능                                │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   격리수준 ↓        정합성 ↓        동시성 ↑        성능 ↑       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
