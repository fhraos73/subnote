#데이터베이스 #동시성제어 #타임스탬프 #필수 #데이터베이스/동시성제어
## 정의
타임스탬프 기반 동시성 제어, Timestamp Ordering
- 각 트랜잭션에 고유한 타임스탬프를 부여하여 실행 순서를 결정하는 동시성 제어 기법
- Lock 없이 타임스탬프 비교만으로 직렬 가능성을 보장하는 비잠금 기반 프로토콜

## 키워드
* TS(Ti): 트랜잭션 Ti의 타임스탬프
* R-TS(X): 데이터 X를 읽은 가장 최근 트랜잭션의 타임스탬프
* W-TS(X): 데이터 X를 쓴 가장 최근 트랜잭션의 타임스탬프
* Thomas Write Rule: 불필요한 쓰기 무시 최적화

## 암기법
* 타임스탬프: "먼저 온 놈이 임자" (작은 TS가 먼저)
* 규칙: "읽쓰타" (R-TS, W-TS, TS 비교)
* 충돌 시: "작으면 Abort" (TS < W-TS/R-TS면 취소)

## 구성요소/특징/유형
### 타임스탬프 규칙
| 연산 | 조건 | 동작 |
|------|------|------|
| Read(X) | TS(Ti) < W-TS(X) | Abort (더 최신 쓰기 존재) |
| Read(X) | TS(Ti) ≥ W-TS(X) | 읽기 허용, R-TS(X) = max(R-TS(X), TS(Ti)) |
| Write(X) | TS(Ti) < R-TS(X) | Abort (더 최신 읽기 존재) |
| Write(X) | TS(Ti) < W-TS(X) | Thomas Rule: 무시 또는 Abort |
| Write(X) | TS(Ti) ≥ R-TS(X) and TS(Ti) ≥ W-TS(X) | 쓰기 허용, W-TS(X) = TS(Ti) |

### Thomas Write Rule
- 조건: TS(Ti) < W-TS(X) 일 때
- 기본: Abort
- Thomas Rule: 쓰기 무시 (어차피 덮어쓰일 값)
- 효과: 불필요한 Abort 감소, 동시성 향상

### 장단점
| 구분 | 내용 |
|------|------|
| 장점 | Deadlock 없음, Lock 오버헤드 없음, 공정한 스케줄링 |
| 단점 | Cascading Rollback 가능, Abort 빈번, 재시작 오버헤드 |

### Lock 기반 vs Timestamp 기반
| 구분 | Lock 기반 (2PL) | Timestamp 기반 |
|------|-----------------|----------------|
| 충돌 해결 | 대기 (Wait) | 재시작 (Abort) |
| Deadlock | 발생 가능 | 발생 안함 |
| 동시성 | Lock 경합 시 저하 | Abort 빈번 시 저하 |
| 구현 | Lock Manager 필요 | Timestamp 관리 필요 |

```
┌─────────────────────────────────────────────────────────────────┐
│                  Timestamp Ordering 동작 원리                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                  타임스탬프 할당                          │   │
│   │                                                         │   │
│   │   T1 ──→ TS(T1) = 100                                   │   │
│   │   T2 ──→ TS(T2) = 150                                   │   │
│   │   T3 ──→ TS(T3) = 200                                   │   │
│   │                                                         │   │
│   │   시작 순서대로 고유 타임스탬프 부여                       │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                  Read 연산 검증                           │   │
│   │                                                         │   │
│   │   Ti가 Read(X) 요청                                      │   │
│   │        │                                                │   │
│   │        ↓                                                │   │
│   │   TS(Ti) < W-TS(X)?                                     │   │
│   │        │                                                │   │
│   │   Yes ─┴─→ Abort Ti (더 최신 쓰기 존재)                  │   │
│   │   No  ───→ Read 허용, R-TS(X) 갱신                       │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                  Write 연산 검증                          │   │
│   │                                                         │   │
│   │   Ti가 Write(X) 요청                                     │   │
│   │        │                                                │   │
│   │        ↓                                                │   │
│   │   TS(Ti) < R-TS(X)?  ──Yes──→ Abort (더 최신 읽기 존재)  │   │
│   │        │                                                │   │
│   │        No                                               │   │
│   │        ↓                                                │   │
│   │   TS(Ti) < W-TS(X)?  ──Yes──→ Thomas Rule: 무시/Abort   │   │
│   │        │                                                │   │
│   │        No                                               │   │
│   │        ↓                                                │   │
│   │   Write 허용, W-TS(X) = TS(Ti)                          │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
