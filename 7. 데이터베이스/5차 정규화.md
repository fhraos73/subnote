#데이터베이스 #정규화 #5NF #조인종속 #필수 #데이터베이스/정규화
## 정의
조인종속 제거, 5NF/PJNF
- 4NF를 만족하면서 조인종속(JD, Join Dependency)을 제거하는 최고 수준의 정규화 단계
- 릴레이션을 분해 후 조인해도 원래 릴레이션과 동일하게 복원되는 무손실 분해 보장
## 키워드
* 조인종속(JD): 릴레이션을 여러 부분으로 분해 후 조인 시 원본 복원
* PJNF(Projection-Join NF): 5NF의 다른 명칭
* 무손실 조인(Lossless Join): 분해 후 조인 시 정보 손실 없음
* 후보키 기반 JD: 모든 JD가 후보키를 통해서만 성립
## 암기법
* "5조후" - 5NF는 조인종속이 후보키 기반
* "분조원" - 분해 후 조인하면 원본 복원
* "피제이" - PJNF = Projection-Join Normal Form
## 구성요소/특징/유형
### 5NF 조건
| 조건 | 설명 |
|------|------|
| 4NF 만족 | 다치종속 없음 |
| 모든 JD가 후보키 기반 | 조인종속이 후보키를 통해서만 성립 |
### 조인종속(JD) 개념
| 특징 | 설명 |
|------|------|
| 정의 | R = R1 ⋈ R2 ⋈ ... ⋈ Rn |
| 표기 | *(R1, R2, ..., Rn) |
| 의미 | 분해된 테이블들의 조인이 원본과 동일 |
### 5NF vs 4NF
| 구분 | 4NF | 5NF |
|------|-----|-----|
| 제거 대상 | 다치종속(MVD) | 조인종속(JD) |
| 분해 단위 | 2개 테이블 | 3개 이상 테이블 |
| 복잡도 | 중간 | 높음 |
### 5NF 적용 시 고려사항
| 항목 | 설명 |
|------|------|
| 실무 적용 | 거의 사용하지 않음 |
| 이유 | 과도한 분해로 조인 비용 증가 |
| 권장 | 일반적으로 3NF 또는 BCNF에서 멈춤 |
## 구성도
```
┌─────────────────────────────────────────────────────────────┐
│                    5NF 변환 과정                             │
├─────────────────────────────────────────────────────────────┤
│  [5NF 위반 - 조인종속 존재]                                   │
│  ┌───────────────────────────────────────────────────────┐ │
│  │ 공급 테이블 (SPJ)                                      │ │
│  │ 공급자(S) │ 부품(P) │ 프로젝트(J)                      │ │
│  ├───────────────────────────────────────────────────────┤ │
│  │    S1    │   P1   │    J1                            │ │
│  │    S1    │   P1   │    J2                            │ │
│  │    S1    │   P2   │    J1                            │ │
│  │    S2    │   P1   │    J1                            │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                             │
│  조인종속 분석:                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 제약조건:                                             │   │
│  │ • S가 P를 공급하고                                    │   │
│  │ • P가 J에 사용되고                                    │   │
│  │ • S가 J에 공급하면                                    │   │
│  │ → S는 P를 J에 공급한다                                │   │
│  │                                                       │   │
│  │ JD: *(SP, PJ, SJ) - 3개 테이블로 분해 가능            │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓ 5NF 적용                         │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐       │
│  │ SP 테이블    │ │ PJ 테이블    │ │ SJ 테이블    │       │
│  ├──────────────┤ ├──────────────┤ ├──────────────┤       │
│  │  S   │  P   │ │  P   │  J   │ │  S   │  J   │       │
│  │  S1  │  P1  │ │  P1  │  J1  │ │  S1  │  J1  │       │
│  │  S1  │  P2  │ │  P1  │  J2  │ │  S1  │  J2  │       │
│  │  S2  │  P1  │ │  P2  │  J1  │ │  S2  │  J1  │       │
│  └──────────────┘ └──────────────┘ └──────────────┘       │
│                                                             │
│  검증: SP ⋈ PJ ⋈ SJ = SPJ (원본 복원)                       │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 정규화 단계 요약                                      │   │
│  │ 1NF: 원자값 │ 2NF: 부분FD │ 3NF: 이행FD              │   │
│  │ BCNF: 결정자=후보키 │ 4NF: MVD │ 5NF: JD             │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```
