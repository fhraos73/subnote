#데이터베이스 #동시성제어 #문제점 #필수 #데이터베이스/동시성제어
## 정의
동시성 미제어 시 이상현상, Concurrency Problems
- 다중 트랜잭션이 동시에 동일 데이터에 접근할 때 제어 없이 발생하는 데이터 불일치 문제
- 트랜잭션의 격리성(Isolation) 미보장으로 인한 데이터 무결성 훼손 현상

## 키워드
* 갱신 분실 (Lost Update): 동시 갱신 시 한쪽 갱신 소실
* 모순성 (Inconsistency): 트랜잭션 중간 데이터 읽기
* 연쇄 복귀 (Cascading Rollback): 의존 트랜잭션 연쇄 취소
* 비완료 의존성 (Uncommitted Dependency): 커밋 안된 데이터 의존
* Dirty Read, Non-Repeatable Read, Phantom Read

## 암기법
* 4대 문제: "갱모연비" (갱신분실, 모순성, 연쇄복귀, 비완료의존성)
* 격리수준 이상현상: "더논팬" (Dirty, Non-repeatable, Phantom)
* 문제 원인: "동시 접근 + 제어 부재 = 불일치"

## 구성요소/특징/유형
### 동시성 제어 문제점
| 문제 | 영문 | 설명 | 예시 |
|------|------|------|------|
| 갱신 분실 | Lost Update | 두 트랜잭션이 동시 갱신, 한쪽 소실 | T1, T2가 동시에 X 수정, T1 결과 소실 |
| 모순성 | Inconsistency | 트랜잭션 중간 상태 읽기 | T1 수행 중 T2가 부분 결과 읽음 |
| 연쇄 복귀 | Cascading Rollback | 한 트랜잭션 취소 시 연쇄 취소 | T1 Abort → T1 결과 읽은 T2도 Abort |
| 비완료 의존성 | Uncommitted Dependency | 커밋 안된 데이터에 의존 | T1 미커밋 데이터를 T2가 읽고 사용 |

### 격리 수준별 이상현상
| 이상현상 | 설명 | 발생 조건 |
|----------|------|-----------|
| Dirty Read | 커밋 안된 데이터 읽기 | Read Uncommitted |
| Non-Repeatable Read | 같은 쿼리 다른 결과 (UPDATE) | Read Committed 이하 |
| Phantom Read | 같은 쿼리 다른 행 수 (INSERT) | Repeatable Read 이하 |

### 문제별 상세 시나리오
| 문제 | 시나리오 |
|------|----------|
| 갱신 분실 | T1: Read(X)→X+10, T2: Read(X)→X+20, T1: Write(X), T2: Write(X) → T1 결과 소실 |
| 모순성 | T1: A-=100, B+=100 수행 중 T2가 A+B 계산 → 합계 불일치 |
| 연쇄 복귀 | T1: Write(X), T2: Read(X), T1: Abort → T2도 Abort 필요 |
