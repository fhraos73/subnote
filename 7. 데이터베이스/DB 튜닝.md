#데이터베이스 #성능최적화 #SQL튜닝 #필수
## 정의
DB 성능 개선을 위한 튜닝
- 데이터베이스 시스템의 응답시간 단축과 처리량 향상을 위한 최적화 활동
- SQL, 인덱스, 설계, 서버 자원 등 다계층 관점의 종합적 성능 개선 기법
## 키워드
* 실행계획(Execution Plan), 옵티마이저, 인덱스 스캔, 풀 테이블 스캔
* 힌트(Hint), 파티셔닝, 정규화/반정규화, I/O 최소화
* 바인드 변수, 조인 최적화, 통계정보
## 암기법
* "설인SQL서" - 설계튜닝, 인덱스튜닝, SQL튜닝, 서버튜닝
* "실옵힌통" - 실행계획, 옵티마이저, 힌트, 통계정보
## 구성요소/특징/유형
### 튜닝 계층별 분류
| 계층 | 튜닝 대상 | 주요 기법 |
|------|----------|----------|
| 설계 튜닝 | 데이터 모델 | 정규화/반정규화, 파티셔닝, 인덱스 설계 |
| SQL 튜닝 | 쿼리문 | 실행계획 분석, 조인 순서 최적화, 힌트 사용 |
| 인덱스 튜닝 | 인덱스 | 인덱스 스캔 효율화, 랜덤 액세스 최소화 |
| 서버 튜닝 | DBMS/OS | 메모리, 버퍼 캐시, 병렬처리 설정 |
### SQL 튜닝 핵심 원칙
- 인덱스 컬럼 가공 금지 (범위 스캔 불가)
- 묵시적 형변환 회피 (명시적 형변환 사용)
- 선행 컬럼 '=' 조건 활용 (인덱스 효율 극대화)
- 불필요한 컬럼 제거 (SELECT * 지양)
### 인덱스 튜닝 핵심 요소
- **인덱스 스캔 효율화**: 선행 컬럼 조건절 최적화
- **랜덤 액세스 최소화**: 테이블 접근 횟수 감소 (성능 영향 더 큼)
