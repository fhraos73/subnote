#데이터베이스 #NewSQL #분산SQL #ACID #필수 #데이터베이스/NoSQL/분산
## 정의
분산 SQL 데이터베이스
- RDBMS의 ACID 트랜잭션과 SQL 인터페이스를 유지하면서 NoSQL의 수평 확장성을 결합한 차세대 데이터베이스
- 분산 아키텍처 기반으로 고성능 트랜잭션 처리와 실시간 분석을 동시에 지원
## 키워드
* ACID + Scale-out: 트랜잭션 보장 + 수평 확장
* 분산 트랜잭션: 여러 노드에 걸친 원자적 트랜잭션
* 자동 샤딩(Auto-Sharding): 자동 데이터 분산
* SQL 호환: 기존 SQL 쿼리 사용 가능
* 대표 제품: Google Spanner, CockroachDB, TiDB
## 암기법
* "뉴에분" - NewSQL = ACID + 분산 확장
* "산에일" - 분산 + ACID + SQL
* "스카티" - Spanner, CockroachDB, TiDB
## 구성요소/특징/유형
### NewSQL 핵심 특징
| 특징 | 설명 |
|------|------|
| ACID 보장 | 분산 환경에서도 트랜잭션 무결성 |
| SQL 지원 | 표준 SQL 쿼리 사용 |
| 수평 확장 | 노드 추가로 성능/용량 확장 |
| 자동 샤딩 | 데이터 자동 분산 및 리밸런싱 |
| 고가용성 | 자동 장애 복구, 무중단 운영 |
### RDBMS vs NoSQL vs NewSQL
| 구분 | RDBMS | NoSQL | NewSQL |
|------|-------|-------|--------|
| 확장 | 수직 | 수평 | 수평 |
| 일관성 | 강한 | 최종 | 강한 |
| 트랜잭션 | ACID | BASE | ACID |
| SQL | 지원 | 제한적 | 지원 |
| 스키마 | 고정 | 유연 | 고정/유연 |
### 주요 NewSQL 제품
| 제품 | 특징 | 사용 사례 |
|------|------|-----------|
| Google Spanner | TrueTime API, 글로벌 분산 | 글로벌 금융 서비스 |
| CockroachDB | PostgreSQL 호환, 자동 복구 | 클라우드 네이티브 앱 |
| TiDB | MySQL 호환, HTAP 지원 | 실시간 분석 |
| VoltDB | 인메모리, 초저지연 | 실시간 트랜잭션 |
| YugabyteDB | PostgreSQL/Cassandra 호환 | 하이브리드 워크로드 |
### NewSQL 아키텍처 구성요소
| 구성요소 | 역할 |
|----------|------|
| SQL Layer | 쿼리 파싱, 최적화, 실행 |
| Transaction Layer | 분산 트랜잭션 관리 |
| Storage Layer | 분산 저장, 복제, 샤딩 |
| Consensus | Raft/Paxos 기반 합의 |
