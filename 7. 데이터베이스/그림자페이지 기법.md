#데이터베이스 #회복기법 #그림자페이지 #필수 #데이터베이스/트랜잭션/회복
## 정의
그림자 복사 회복, Shadow Paging
- 트랜잭션 수행 시 현재 페이지 테이블과 그림자 페이지 테이블 두 개를 유지하는 회복 기법
- 장애 발생 시 그림자 페이지 테이블로 즉시 복구하여 Undo/Redo 로그 불필요

## 키워드
* Current Page Table: 트랜잭션이 변경하는 현재 페이지 테이블
* Shadow Page Table: 트랜잭션 시작 시점의 복사본 (변경 불가)
* No-Undo/No-Redo: 로그 기반 회복 연산 불필요
* Commit: Current → Shadow 교체로 원자적 완료

## 암기법
* 그림자: "그림자는 변하지 않는다" (Shadow는 원본 유지)
* 회복: "노언노리" (No-Undo, No-Redo)
* Commit: "커밋은 포인터만 바꿔" (페이지 테이블 포인터 교체)

## 구성요소/특징/유형
### 동작 원리
| 단계 | 동작 | 설명 |
|------|------|------|
| 시작 | Shadow 생성 | Current Page Table 복사 |
| 수행 | Current 수정 | 변경은 Current에만 반영 |
| Commit | 포인터 교체 | Current → 새 Shadow로 지정 |
| Abort | Current 폐기 | Shadow로 즉시 복구 |

### 장단점 비교
| 구분 | 내용 |
|------|------|
| 장점 | Undo/Redo 로그 불필요, 회복 단순, Abort 시 즉시 복구 |
| 단점 | 페이지 단편화, 디스크 I/O 증가, 동시성 제어 어려움 |
| 적용 | 소규모 DB, 단일 사용자 환경, 임베디드 시스템 |

### 로그 기반 vs 그림자 페이지
| 구분 | 로그 기반 회복 | 그림자 페이지 |
|------|----------------|---------------|
| 회복 연산 | Undo/Redo 필요 | 불필요 |
| 저장 공간 | 로그 파일 | 이중 페이지 테이블 |
| 동시성 | 지원 용이 | 지원 어려움 |
| 회복 시간 | 로그 분석 필요 | 즉시 복구 |
| 실용성 | 대규모 DBMS 표준 | 제한적 사용 |

```
│            │                               │                    │
│            ↓                               ↓                    │
│   ┌──────────────────┐            ┌──────────────────┐         │

│   └──────────────────┘            └──────────────────┘         │

│   ┌─────────────────────────────────────────────────────────┐   │
│   │                      Commit 시                           │   │
│   │                                                         │   │
│   │    DB Pointer ─────→ Current Page Table                 │   │
│   │                      (Current가 새 Shadow가 됨)          │   │

│   │                      Abort 시                           │   │
│   │                                                         │   │
│   │    DB Pointer ─────→ Shadow Page Table (원본 유지)       │   │
│   │                      (Current 폐기)                      │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
