#데이터베이스 #마이그레이션 #오픈소스 #필수
## 정의
데이터베이스 시스템 교체, DBMS 전환
- 기존 DBMS에서 새로운 DBMS로 데이터와 애플리케이션을 이전하는 프로젝트
- 오픈소스 DBMS 전환, 클라우드 DB 전환, 벡터 DB 전환 등 포함
## 키워드
* 마이그레이션, 스키마 변환, 데이터 이관, 호환성, TCO
## 암기법
* "분설이검운" - 분석, 설계, 이관, 검증, 운영
## 전환 유형
| 유형 | 설명 | 사례 |
|------|------|------|
| 상용→오픈소스 | 라이선스 비용 절감 | Oracle→PostgreSQL |
| 온프레미스→클라우드 | 클라우드 전환 | MySQL→Aurora |
| RDBMS→NoSQL | 비정형 데이터 처리 | Oracle→MongoDB |
| RDBMS→Vector DB | AI/ML 워크로드 | PostgreSQL→Pinecone |
## 전환 절차
```
[현황분석] → [전환설계] → [데이터이관] → [검증테스트] → [운영전환]
 AS-IS분석    TO-BE설계    ETL/CDC      기능/성능     병행운영
 호환성분석   스키마변환   데이터검증   회귀테스트   컷오버
```
## 구성도
```
┌──────────────┐         ┌──────────────┐
│  Source DB   │ ──ETL─→ │  Target DB   │
│  (Oracle)    │         │ (PostgreSQL) │
└──────────────┘         └──────────────┘
       │                        │
       └────── 데이터 검증 ──────┘
```
## 고려사항
| 항목 | 고려사항 |
|------|----------|
| 호환성 | SQL 문법, 함수, 데이터타입 차이 |
| 성능 | 실행계획, 인덱스 전략 재검토 |
| 애플리케이션 | 드라이버, 커넥션 풀 변경 |
| 운영 | 백업/복구, 모니터링 체계 |
