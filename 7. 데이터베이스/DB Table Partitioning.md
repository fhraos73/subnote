#데이터베이스 #성능 #파티셔닝 #필수 #데이터베이스/NoSQL/분산
## 정의
대용량 테이블 분할 관리, 파티셔닝
- 대용량 테이블을 작은 단위로 분할하여 성능과 관리 효율성을 높이는 기법
- 논리적으로 하나의 테이블이지만 물리적으로 여러 파티션으로 저장
## 키워드
* Range, List, Hash, Composite, 파티션 키, 파티션 프루닝
## 암기법
* "레리해컴" - Range, List, Hash, Composite
## 파티셔닝 유형
| 유형 | 설명 | 적용 사례 |
|------|------|-----------|
| Range | 범위 기준 분할 | 날짜별 데이터 |
| List | 특정 값 목록 기준 | 지역, 부서 코드 |
| Hash | 해시 함수 기준 | 균등 분배 필요 시 |
| Composite | 복합 파티셔닝 | Range-Hash 조합 |
## 구성도
```
┌─────────────────────────────────────────┐
│           논리적 테이블 (SALES)          │
├─────────────────────────────────────────┤
│ ┌─────────┐ ┌─────────┐ ┌─────────┐    │
│ │ 2024_Q1 │ │ 2024_Q2 │ │ 2024_Q3 │    │
│ │ (Range) │ │ (Range) │ │ (Range) │    │
│ └─────────┘ └─────────┘ └─────────┘    │
└─────────────────────────────────────────┘
```
## 파티셔닝 장점
| 장점 | 설명 |
|------|------|
| 성능 향상 | 파티션 프루닝으로 스캔 범위 축소 |
| 관리 용이 | 파티션 단위 백업/삭제/이동 |
| 가용성 | 파티션 단위 장애 격리 |
| 병렬 처리 | 파티션별 병렬 쿼리 실행 |
## 파티션 프루닝
```
SELECT * FROM SALES WHERE sale_date = '2024-01-15'
→ 2024_Q1 파티션만 스캔 (나머지 파티션 제외)
```
