#데이터베이스 #정규화 #3NF #필수
## 정의
이행함수종속 제거, 3NF
- 2NF를 만족하면서 기본키가 아닌 모든 속성이 기본키에 이행적으로 종속되지 않는 정규화 단계
- 비키 속성 간의 함수적 종속 관계를 분리하여 독립적인 테이블로 구성
## 키워드
* 이행함수종속(Transitive FD): X→Y, Y→Z ⇒ X→Z
* 비키 속성(Non-key Attribute): 기본키가 아닌 속성
* 결정자(Determinant): 다른 속성을 결정하는 속성
* 직접 종속: 기본키에 직접적으로 종속
## 암기법
* "3이직" - 3NF는 이행종속 제거하여 직접종속만 남김
* "비비분" - 비키→비키 종속 분리
* "이행제삼" - 이행적 종속을 제3정규화로 제거
## 구성요소/특징/유형
### 3NF 조건
| 조건 | 설명 |
|------|------|
| 2NF 만족 | 부분함수종속 없음 |
| 이행함수종속 없음 | 비키 속성이 다른 비키 속성에 종속되지 않음 |
| 직접 종속 | 모든 비키 속성이 기본키에만 직접 종속 |
### 이행함수종속 식별
| 단계 | 설명 |
|------|------|
| 종속성 나열 | 모든 함수적 종속성 파악 |
| 이행성 확인 | A→B, B→C 형태의 종속 관계 탐색 |
| 분리 결정 | B→C 관계를 별도 테이블로 분리 |
### 3NF vs BCNF
| 구분 | 3NF | BCNF |
|------|-----|------|
| 조건 | 이행종속 제거 | 모든 결정자가 후보키 |
| 강도 | 약함 | 강함 |
| 무손실 분해 | 항상 보장 | 보장 안 될 수 있음 |
## 구성도
```
┌─────────────────────────────────────────────────────────────┐
│                    3NF 변환 과정                             │
├─────────────────────────────────────────────────────────────┤
│  [3NF 위반 - 이행함수종속 존재]                               │
│  ┌───────────────────────────────────────────────────────┐ │
│  │ 학생 테이블                                            │ │
│  │ 학번(PK) │ 이름 │ 학과코드 │ 학과명 │ 학과장           │ │
│  ├───────────────────────────────────────────────────────┤ │
│  │   001   │홍길동│  D01   │ 컴공  │ 김교수            │ │
│  │   002   │김철수│  D01   │ 컴공  │ 김교수            │ │
│  │   003   │이영희│  D02   │ 전자  │ 박교수            │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                             │
│  함수적 종속성 분석:                                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 학번 → 이름, 학과코드           (직접 종속) ✓         │   │
│  │ 학과코드 → 학과명, 학과장        (비키→비키) ✗         │   │
│  │ 학번 → 학과코드 → 학과명, 학과장  (이행 종속) ✗         │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓ 3NF 적용                         │
│  ┌───────────────────────────┐  ┌─────────────────────────┐│
│  │ 학생 테이블                │  │ 학과 테이블              ││
│  ├───────────────────────────┤  ├─────────────────────────┤│
│  │ 학번(PK) │ 이름 │ 학과코드 │  │학과코드(PK)│학과명│학과장││
│  │   001   │홍길동│  D01   │  │   D01   │ 컴공 │김교수││
│  │   002   │김철수│  D01   │  │   D02   │ 전자 │박교수││
│  │   003   │이영희│  D02   │  └─────────────────────────┘│
│  └───────────────────────────┘                             │
│                                                             │
│  이행종속 제거 효과:                                         │
│  • 학과명/학과장 변경 시 한 곳만 수정 (갱신이상 해결)          │
│  • 학과 정보 독립적 관리 가능 (삽입/삭제이상 해결)             │
└─────────────────────────────────────────────────────────────┘
```
