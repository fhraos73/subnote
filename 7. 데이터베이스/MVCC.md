#데이터베이스 #동시성제어 #MVCC #다중버전 #필수 #데이터베이스/동시성제어
## 정의
다중 버전 동시성 제어, MVCC
- Multi-Version Concurrency Control
- 데이터 항목의 여러 버전을 유지하여 읽기와 쓰기 연산 간 충돌을 최소화하는 동시성 제어 기법
- 읽기 연산이 쓰기 연산을 블로킹하지 않아 높은 동시성 제공

## 키워드
* 다중 버전: 데이터의 여러 시점 버전 유지
* Snapshot Isolation: 트랜잭션 시작 시점의 스냅샷 읽기
* Read Timestamp, Write Timestamp: 버전별 타임스탬프
* Garbage Collection: 불필요한 구버전 정리

## 암기법
* MVCC: "엠브씨씨는 버전 여러 개" (Multi-Version)
* 핵심: "읽쓰분리" (읽기와 쓰기 분리로 블로킹 없음)
* 장점: "읽기는 기다리지 않아" (Read Never Blocks)

## 구성요소/특징/유형
### MVCC 버전 구조
| 구성요소 | 설명 |
|----------|------|
| 데이터 버전 | <X, Value, W-TS, R-TS> |
| W-TS | 해당 버전을 생성한 트랜잭션의 타임스탬프 |
| R-TS | 해당 버전을 읽은 가장 최근 트랜잭션의 타임스탬프 |
| Version Chain | 동일 데이터의 버전들을 연결한 리스트 |

### MVCC 연산 규칙
| 연산 | 동작 |
|------|------|
| Read(X) | TS(Ti) 이하의 가장 최근 버전 반환 |
| Write(X) | 새 버전 생성 (기존 버전 유지) |

### MVCC 구현 방식
| 방식 | 설명 | 예시 DBMS |
|------|------|-----------|
| MGA (Multi Generation Architecture) | Undo 영역에 구버전 저장 | Oracle |
| Append-Only | 새 버전을 새 위치에 저장 | PostgreSQL |
| In-Place Update | 현재 위치 갱신 + Undo Log | MySQL InnoDB |

### 장단점
| 구분 | 내용 |
|------|------|
| 장점 | 읽기-쓰기 비블로킹, 높은 동시성, Deadlock 감소 |
| 단점 | 저장 공간 증가, Garbage Collection 오버헤드, Write Skew 가능 |

### MVCC vs Lock 기반
| 구분 | MVCC | Lock 기반 |
|------|------|-----------|
| 읽기-쓰기 충돌 | 없음 (다른 버전 읽기) | 대기 발생 |
| 저장 공간 | 다중 버전으로 증가 | 단일 버전 |
| 동시성 | 높음 | 상대적 낮음 |
| 구현 복잡도 | 높음 | 낮음 |

## 구성도
```
┌─────────────────────────────────────────────────────────────────┐
│                      MVCC 동작 원리                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                  다중 버전 구조                           │   │
│   │                                                         │   │
│   │   데이터 X의 버전 체인:                                   │   │
│   │                                                         │   │
│   │   ┌─────────┐   ┌─────────┐   ┌─────────┐              │   │
│   │   │ X(v3)   │──→│ X(v2)   │──→│ X(v1)   │              │   │
│   │   │ Val=300 │   │ Val=200 │   │ Val=100 │              │   │
│   │   │ W-TS=150│   │ W-TS=100│   │ W-TS=50 │              │   │
│   │   └─────────┘   └─────────┘   └─────────┘              │   │
│   │      최신           ↑            구버전                  │   │
│   │                     │                                   │   │
│   │                  TS=120인 트랜잭션은                      │   │
│   │                  v2(W-TS=100) 읽기                       │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                  읽기-쓰기 비블로킹                        │   │
│   │                                                         │   │
│   │   T1(TS=100): Read(X) ──────────────────────────────    │   │
│   │                    │                                    │   │
│   │   T2(TS=150):      │    Write(X) ──→ 새 버전 생성       │   │
│   │                    │         │                          │   │
│   │                    ↓         ↓                          │   │
│   │               X(v1) 읽기  X(v2) 생성                     │   │
│   │                                                         │   │
│   │   → T1은 T2의 쓰기를 기다리지 않고 구버전 읽기            │   │
│   │   → T2는 T1의 읽기를 기다리지 않고 새 버전 생성           │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  Garbage Collection: 모든 활성 트랜잭션보다 오래된        │   │
│   │                      버전은 삭제 가능                     │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
