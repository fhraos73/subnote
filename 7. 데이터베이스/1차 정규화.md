#데이터베이스 #정규화 #1NF #필수
## 정의
원자값 보장, 1NF
- 릴레이션의 모든 속성이 원자값(Atomic Value)만을 가지도록 하는 정규화 단계
- 반복 그룹(Repeating Group)을 제거하여 테이블의 기본 구조를 확립
## 키워드
* 원자값(Atomic Value): 더 이상 분해 불가능한 단일 값
* 반복 그룹(Repeating Group): 하나의 속성에 여러 값 존재
* 다중값 속성(Multi-valued Attribute): 복수의 값을 가지는 속성
* 도메인(Domain): 속성이 가질 수 있는 값의 집합
## 암기법
* "원반제" - 원자값 보장, 반복그룹 제거
* "1원자" - 1NF는 원자값
* "더쪼못" - 더 이상 쪼개지 못하는 값
## 구성요소/특징/유형
### 1NF 조건
| 조건 | 설명 |
|------|------|
| 원자값 | 모든 속성 값이 단일 값 |
| 반복그룹 제거 | 동일 속성의 반복 제거 |
| 기본키 존재 | 각 행을 유일하게 식별 |
### 1NF 위반 사례
| 위반 유형 | 예시 | 해결 방법 |
|-----------|------|-----------|
| 다중값 속성 | 전화번호: "010-1234, 010-5678" | 별도 테이블 분리 |
| 반복 그룹 | 과목1, 과목2, 과목3 컬럼 | 행으로 분리 |
| 복합 속성 | 주소: "서울시 강남구 역삼동" | 시/구/동 분리 |
### 1NF 변환 방법
1. 다중값을 가진 속성을 별도 행으로 분리
2. 반복되는 컬럼 그룹을 별도 테이블로 분리
3. 분리된 테이블에 외래키 설정
