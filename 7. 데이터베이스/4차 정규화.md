#데이터베이스 #정규화 #4NF #다치종속 #필수 #데이터베이스/정규화
## 정의
다치종속 제거, 4NF
- BCNF를 만족하면서 다치종속(MVD, Multi-Valued Dependency)을 제거하는 정규화 단계
- 하나의 속성이 다른 속성에 대해 독립적으로 여러 값을 가지는 관계를 분리
## 키워드
* 다치종속(MVD): A→→B로 표기, 하나의 A값에 여러 B값이 독립적으로 대응
* 비정상 다치종속(Non-trivial MVD): 분해가 필요한 다치종속
* 무손실 분해(Lossless Decomposition): 정보 손실 없는 테이블 분리
* 독립적 다중값: 서로 관련 없는 복수의 값
## 암기법
* "4다제" - 4NF는 다치종속 제거
* "독다분" - 독립적 다중값을 분리
* "쌍화살" - A→→B (쌍화살표로 다치종속 표기)
## 구성요소/특징/유형
### 4NF 조건
| 조건 | 설명 |
|------|------|
| BCNF 만족 | 모든 결정자가 후보키 |
| 비정상 MVD 없음 | 독립적인 다중값 속성이 분리됨 |
### 다치종속(MVD) 특징
| 특징 | 설명 |
|------|------|
| 표기법 | A→→B (A가 B를 다치 결정) |
| 발생 조건 | 최소 3개 속성, 2개가 독립적으로 다중값 |
| 문제점 | 불필요한 튜플 중복 발생 |
### 다치종속 vs 함수종속
| 구분 | 함수종속(FD) | 다치종속(MVD) |
|------|-------------|--------------|
| 표기 | A→B | A→→B |
| 관계 | 1:1 결정 | 1:N 결정 |
| 값 개수 | 단일값 | 다중값 |
### 4NF 위반 식별
- 한 속성에 대해 독립적인 다중값 속성이 2개 이상 존재
- 모든 조합이 튜플로 존재해야 하는 상황
## 구성도
```
┌─────────────────────────────────────────────────────────────┐
│                    4NF 변환 과정                             │
├─────────────────────────────────────────────────────────────┤
│  [4NF 위반 - 다치종속 존재]                                   │
│  ┌───────────────────────────────────────────────────────┐ │
│  │ 강의 테이블                                            │ │
│  │ 과목(PK) │ 교수(PK) │ 교재(PK)                         │ │
│  ├───────────────────────────────────────────────────────┤ │
│  │   DB    │  김교수  │ DB개론                           │ │
│  │   DB    │  김교수  │ SQL완성                          │ │
│  │   DB    │  박교수  │ DB개론                           │ │
│  │   DB    │  박교수  │ SQL완성                          │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                             │
│  다치종속 분석:                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 과목 →→ 교수  (DB 과목에 김교수, 박교수)               │   │
│  │ 과목 →→ 교재  (DB 과목에 DB개론, SQL완성)              │   │
│  │                                                       │   │
│  │ 교수와 교재는 서로 독립적 (모든 조합 필요 = 중복!)       │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓ 4NF 적용                         │
│  ┌─────────────────────────┐  ┌─────────────────────────┐ │
│  │ 과목_교수 테이블         │  │ 과목_교재 테이블         │ │
│  ├─────────────────────────┤  ├─────────────────────────┤ │
│  │ 과목(PK) │ 교수(PK)     │  │ 과목(PK) │ 교재(PK)     │ │
│  │   DB    │   김교수     │  │   DB    │  DB개론      │ │
│  │   DB    │   박교수     │  │   DB    │  SQL완성     │ │
│  └─────────────────────────┘  └─────────────────────────┘ │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 4NF 효과                                             │   │
│  │ • 중복 튜플 제거 (4행 → 2행 + 2행)                    │   │
│  │ • 독립적인 다중값 관계를 별도 관리                     │   │
│  │ • 삽입/삭제 이상 해결                                 │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```
