#데이터베이스 #반정규화 #성능최적화 #필수 #데이터베이스/정규화
## 정의
성능 향상 위한 역정규화 (Denormalization)
- 정규화된 데이터 모델에서 조회 성능 향상을 위해 의도적으로 데이터 중복을 허용하는 기법
- 조인 연산 감소, 디스크 I/O 최소화를 통해 읽기 성능을 개선하는 데이터 모델링 기법
## 키워드
* 테이블 병합: 1:1, 1:M 관계 테이블 통합
* 테이블 분할: 수직분할(속성), 수평분할(파티셔닝)
* 중복 컬럼: JOIN 회피를 위한 컬럼 복제
* 파생 컬럼: 계산 결과를 미리 저장
* 데이터 정합성: 중복으로 인한 불일치 위험
## 암기법
* "반테컬관" - 반정규화 = 테이블/컬럼/관계 반정규화
* "병분추" - 병합/분할/추가
* "조성향" - 조회 성능 향상 목적
## 구성요소/특징/유형
### 반정규화 유형
| 구분 | 기법 | 설명 |
|------|------|------|
| 테이블 | 병합 | 1:1, 1:M, 슈퍼/서브타입 병합 |
| 테이블 | 분할 | 수직분할(컬럼), 수평분할(행) |
| 테이블 | 추가 | 중복/통계/이력/부분 테이블 |
| 컬럼 | 중복 | JOIN 대상 컬럼 복제 |
| 컬럼 | 파생 | 계산값 미리 저장 |
| 컬럼 | 이력 | 최신값 별도 저장 |
| 관계 | 중복 | 조인 경로 단축 |
### 반정규화 장단점
| 장점 | 단점 |
|------|------|
| 조회(SELECT) 성능 향상 | 입력/수정/삭제 성능 저하 |
| 조인 연산 감소 | 데이터 정합성 문제 |
| 디스크 I/O 감소 | 저장 공간 증가 |
| 쿼리 단순화 | 데이터 무결성 관리 필요 |
### 반정규화 적용 절차
| 단계 | 설명 |
|------|------|
| 1. 대상 조사 | 성능 저하 테이블 식별 |
| 2. 대안 검토 | 인덱스, 클러스터링, 파티션 검토 |
| 3. 반정규화 수행 | 적절한 기법 적용 |
| 4. 검증 | 성능 테스트 및 모니터링 |
## 구성도
```
┌─────────────────────────────────────────────────────────────┐
│                    반정규화 기법 유형                           │
├─────────────────────────────────────────────────────────────┤
│  [1. 테이블 반정규화]                                           │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 테이블 병합                                            │   │
│  │ ┌─────┐  ┌─────┐         ┌───────────────┐          │   │
│  │ │ 주문 │──│ 배송 │  →      │ 주문_배송        │          │   │
│  │ └─────┘  └─────┘   1:1   └───────────────┘          │   │
│  │                                                       │   │
│  │ 테이블 분할                                           │   │
│  │ ┌───────────────┐      ┌─────────┐ ┌─────────┐     │   │
│  │ │ 고객(전체)    │  →   │고객_기본│ │고객_상세│     │   │
│  │ └───────────────┘ 수직 └─────────┘ └─────────┘     │   │
│  │                                                       │   │
│  │ ┌───────────────┐      ┌─────────┐ ┌─────────┐     │   │
│  │ │ 주문(전체)    │  →   │주문_2024│ │주문_2025│     │   │
│  │ └───────────────┘ 수평 └─────────┘ └─────────┘     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  [2. 컬럼 반정규화]                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 중복 컬럼 추가                                        │   │
│  │ ┌─────────────────────────────────────────────┐     │   │
│  │ │ 주문: 주문ID, 고객ID, 고객명(중복), 상품ID         │     │   │
│  │ └─────────────────────────────────────────────┘     │   │
│  │ → 고객 테이블 JOIN 없이 고객명 조회 가능                   │   │
│  │                                                       │   │
│  │ 파생 컬럼 추가                                        │   │
│  │ ┌─────────────────────────────────────────────┐     │   │
│  │ │ 주문: 주문ID, 수량, 단가, 총액(파생)         │     │   │
│  │ └─────────────────────────────────────────────┘     │   │
│  │ → 매번 수량*단가 계산 불필요                          │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  [3. 관계 반정규화]                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 중복 관계 추가                                        │   │
│  │ 주문 ─→ 주문상세 ─→ 상품                              │   │
│  │   └────────────────→ (중복 관계로 직접 연결)          │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  [적용 판단 기준]                                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 조회 빈도 높음 + 갱신 빈도 낮음 → 반정규화 적합        │   │
│  │ 조회 빈도 낮음 + 갱신 빈도 높음 → 정규화 유지          │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```
