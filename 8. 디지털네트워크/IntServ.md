#디지털네트워크 #QoS #IntServ #필수
## 정의
통합 서비스(IntServ)
- RSVP 시그널링을 통해 End-to-End QoS를 보장하는 Fine-grained QoS 아키텍처
- 플로우별 자원 예약으로 지연, 대역폭, 손실률을 보장하는 RFC 1633 기반 모델
## 키워드
* RSVP, PATH, RESV, Soft State
* TSPEC, RSPEC, Flow Spec
* Guaranteed Service, Controlled Load
## 암기법
* "인알플" - IntServ, RSVP, Flow별 예약
* "패리소" - PATH, RESV, Soft State
## 구성요소/특징/유형
| 구분 | 설명 | 비고 |
|------|------|------|
| RSVP | 자원 예약 시그널링 프로토콜 | PATH/RESV 메시지 |
| PATH | 송신→수신, 경로 탐색 | 라우터 경로 기록 |
| RESV | 수신→송신, 자원 예약 요청 | 역방향 전파 |
| Soft State | 주기적 갱신, 타임아웃 해제 | 명시적 해제 불필요 |
| TSPEC | Traffic Spec, 트래픽 특성 정의 | Token Bucket 파라미터 |
| RSPEC | Request Spec, QoS 요구사항 | 지연, 대역폭 요청 |
## 구성도
```
┌─────────────────────────────────────────────────────────┐
│                IntServ/RSVP 동작 흐름                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   송신자                라우터들                수신자  │
│   ┌────┐    PATH     ┌────┐    PATH     ┌────┐        │
│   │ S  │ ──────────> │ R1 │ ──────────> │ D  │        │
│   │    │             │    │             │    │        │
│   │    │    RESV     │    │    RESV     │    │        │
│   │    │ <────────── │    │ <────────── │    │        │
│   └────┘             └────┘             └────┘        │
│     │                  │                  │           │
│     └──────────────────┴──────────────────┘           │
│              자원 예약 완료 (E2E QoS)                  │
├─────────────────────────────────────────────────────────┤
│                서비스 클래스                            │
│  ┌─────────────────┬────────────────────────────────┐ │
│  │ Guaranteed      │ 대역폭, 지연, 손실 완전 보장   │ │
│  │ Controlled Load │ 경부하 네트워크 수준 서비스    │ │
│  │ Best Effort     │ QoS 보장 없음 (기본)           │ │
│  └─────────────────┴────────────────────────────────┘ │
├─────────────────────────────────────────────────────────┤
│  한계: 플로우별 상태 유지 → 확장성 문제 (백본 부적합) │
└─────────────────────────────────────────────────────────┘
```
