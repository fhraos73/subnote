#디지털네트워크 #로드밸런싱 #스위칭 #옵션
## 정의
전송 계층 기반 스위칭, L4 스위치
- OSI 4계층(전송 계층)의 TCP/UDP 포트 정보를 기반으로 트래픽을 분배하는 네트워크 장비
- 서버 로드밸런싱, 방화벽, QoS 기능을 수행하여 서비스 가용성 향상
## 키워드
* TCP/UDP 포트, 로드밸런싱, VIP, Real Server, Health Check, Session Persistence
## 암기법
* "포로비리헬" - 포트, 로드밸런싱, VIP, Real Server, Health Check
## 구성요소/특징/유형
| 구분 | 설명 | 비고 |
|------|------|------|
| VIP | Virtual IP, 가상 서비스 IP | 클라이언트 접속점 |
| Real Server | 실제 서비스 서버 | 부하 분산 대상 |
| Health Check | 서버 상태 점검 | 장애 감지 |
| Session Persistence | 세션 유지 | Sticky Session |
### 로드밸런싱 알고리즘
| 알고리즘 | 설명 | 특징 |
|----------|------|------|
| Round Robin | 순차적 분배 | 단순, 균등 |
| Weighted RR | 가중치 기반 분배 | 서버 성능 고려 |
| Least Connection | 연결 수 최소 서버 | 동적 분배 |
| Hash | IP/Port 해시 | 세션 유지 |
| Response Time | 응답 시간 기반 | 성능 최적화 |
### L4 vs L7 스위치 비교
| 구분 | L4 스위치 | L7 스위치 |
|------|-----------|-----------|
| 계층 | 전송 계층 | 응용 계층 |
| 기준 | IP, Port | URL, Cookie, Header |
| 성능 | 고속 | 상대적 저속 |
| 기능 | 단순 분배 | 콘텐츠 기반 분배 |
| 보안 | 기본 | WAF 가능 |
## 구성도
```
┌─────────────────────────────────────────────────────────────┐
│                    L4 스위치 구성도                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [클라이언트 요청]                                           │
│       │                                                     │
│       │  요청: VIP:80 (예: 10.0.0.100:80)                   │
│       ↓                                                     │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                    L4 스위치                          │    │
│  │  ┌─────────────────────────────────────────────┐    │    │
│  │  │           로드밸런싱 알고리즘                  │    │    │
│  │  │  (Round Robin / Least Connection / Hash)    │    │    │
│  │  └─────────────────────────────────────────────┘    │    │
│  │                                                      │    │
│  │  ┌─────────────────────────────────────────────┐    │    │
│  │  │              Health Check                    │    │    │
│  │  │  (TCP Connect / HTTP GET / ICMP Ping)       │    │    │
│  │  └─────────────────────────────────────────────┘    │    │
│  └─────────────────────────────────────────────────────┘    │
│       │              │              │                       │
│       ↓              ↓              ↓                       │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐                   │
│  │ Server1 │   │ Server2 │   │ Server3 │                   │
│  │10.0.0.1 │   │10.0.0.2 │   │10.0.0.3 │                   │
│  │ :80     │   │ :80     │   │ :80     │                   │
│  └─────────┘   └─────────┘   └─────────┘                   │
│                                                             │
│  [NAT 모드 vs DSR 모드]                                     │
│                                                             │
│  NAT 모드:                                                  │
│  Client → L4(VIP) → Server → L4 → Client                   │
│  (양방향 트래픽이 L4 통과)                                   │
│                                                             │
│  DSR(Direct Server Return) 모드:                            │
│  Client → L4(VIP) → Server → Client                        │
│  (응답은 L4 우회, 고성능)                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
