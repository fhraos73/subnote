#디지털네트워크 #데이터링크 #OSI #옵션
## 정의
인접 노드 간 프레임 전송 계층, Data Link Layer
- OSI 7계층 중 2계층으로 물리적 주소(MAC) 기반 프레임 전송 담당
- 오류 검출/수정, 흐름 제어, 매체 접근 제어 수행
## 키워드
* MAC, LLC, 프레임, 스위치, 브리지, CSMA/CD, 오류제어, 흐름제어
## 암기법
* 데이터링크 기능: 프오흐매(프레이밍, 오류제어, 흐름제어, 매체접근제어)
* 부계층: 엘맥(LLC-논리링크제어, MAC-매체접근제어)
## 구성요소
| 구분 | 설명 | 비고 |
|------|------|------|
| LLC | 논리 링크 제어 | 상위 계층 인터페이스 |
| MAC | 매체 접근 제어 | 물리적 주소, 충돌 제어 |
| 프레임 | 데이터링크 계층 PDU | 헤더+데이터+트레일러 |
| 스위치 | L2 스위칭 장비 | MAC 테이블 기반 |
| 브리지 | 세그먼트 연결 장비 | 충돌 도메인 분리 |
## 주요 기능
| 기능 | 설명 | 기법 |
|------|------|------|
| 프레이밍 | 비트 스트림을 프레임으로 구성 | 시작/종료 플래그 |
| 오류제어 | 오류 검출 및 재전송 | CRC, ARQ |
| 흐름제어 | 송수신 속도 조절 | Stop-and-Wait, 슬라이딩윈도우 |
| 매체접근제어 | 공유 매체 접근 관리 | CSMA/CD, CSMA/CA |
| 주소지정 | 물리적 주소 사용 | MAC 주소 (48bit) |
## 구성도
```
[데이터링크 계층 구조]
┌─────────────────────────────────────┐
│     Network Layer (3)               │
├─────────────────────────────────────┤
│     Data Link Layer (2)             │
│  ┌─────────────────────────────────┐│
│  │    LLC (Logical Link Control)   ││
│  │    - 상위 계층 인터페이스        ││
│  │    - 흐름/오류 제어              ││
│  ├─────────────────────────────────┤│
│  │    MAC (Media Access Control)   ││
│  │    - MAC 주소 관리               ││
│  │    - 매체 접근 제어              ││
│  └─────────────────────────────────┘│
├─────────────────────────────────────┤
│     Physical Layer (1)              │
└─────────────────────────────────────┘

[프레임 구조]
┌────────┬────────┬─────────┬────────┬─────────┐
│Preamble│Dest MAC│Source MAC│  Data  │   FCS   │
│ (동기) │(목적지)│ (출발지) │(페이로드)│ (CRC)  │
└────────┴────────┴─────────┴────────┴─────────┘
   8B       6B        6B      46~1500B    4B

[매체접근제어 방식]
경쟁방식           비경쟁방식
├─ CSMA/CD         ├─ Token Ring
│  (유선 LAN)      ├─ Token Bus
└─ CSMA/CA         └─ Polling
   (무선 LAN)
```
