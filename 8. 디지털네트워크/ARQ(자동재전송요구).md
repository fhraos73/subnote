#디지털네트워크 #오류제어 #데이터링크 #옵션
## 정의
오류 시 재전송 요구, ARQ
- Automatic Repeat reQuest, 오류 검출 시 송신측에 재전송을 요청하는 오류 제어 기법
- 수신측에서 오류를 검출하면 NAK를 보내 해당 프레임의 재전송을 요구
## 키워드
* Stop-and-Wait, Go-Back-N, Selective Repeat, ACK, NAK, 타임아웃, 슬라이딩 윈도우
## 암기법
* "스고셀" - Stop-and-Wait, Go-Back-N, Selective Repeat
## 구성요소/특징/유형
| 구분 | 설명 | 장단점 |
|------|------|--------|
| Stop-and-Wait | 한 프레임씩 전송 후 ACK 대기 | 단순, 효율 낮음 |
| Go-Back-N | 오류 발생 지점부터 모두 재전송 | 구현 단순, 대역폭 낭비 |
| Selective Repeat | 오류 프레임만 선택적 재전송 | 효율적, 구현 복잡 |
### ARQ 유형 비교
| 구분 | Stop-and-Wait | Go-Back-N | Selective Repeat |
|------|---------------|-----------|------------------|
| 윈도우 크기 | 1 | N | N |
| 재전송 범위 | 1개 | N개 이상 | 1개 |
| 버퍼 요구 | 최소 | 송신측 | 양측 |
| 효율성 | 낮음 | 중간 | 높음 |
| 구현 복잡도 | 단순 | 중간 | 복잡 |
## 구성도
```
┌─────────────────────────────────────────────────────────────┐
│                    ARQ 유형별 동작                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [Stop-and-Wait ARQ]                                        │
│  송신: F0 ──────→ (대기) ──────→ F1 ──────→ (대기)          │
│  수신:    ←── ACK0 ──────────────── ←── ACK1               │
│                                                             │
│  [Go-Back-N ARQ] (N=4, 프레임 2 오류)                       │
│  송신: F0 F1 F2 F3 F4 ──→ F2 F3 F4 F5 (2부터 재전송)        │
│  수신: A0 A1  ✗  ──────→ A2 A3 A4 A5                       │
│                                                             │
│  [Selective Repeat ARQ] (프레임 2 오류)                     │
│  송신: F0 F1 F2 F3 F4 ──→ F2 (2만 재전송)                   │
│  수신: A0 A1 NAK A3 A4 → A2                                │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                    ARQ 처리 흐름                             │
│                                                             │
│  ┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐              │
│  │ 전송  │ →  │ 오류  │ →  │ 검출  │ →  │재전송 │              │
│  │      │    │ 발생  │    │(CRC) │    │ 요청  │              │
│  └──────┘    └──────┘    └──────┘    └──────┘              │
│       ↑                                    │                │
│       └────────────────────────────────────┘                │
│                    재전송                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
