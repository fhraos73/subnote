#디지털네트워크 #QoS #트래픽폴리싱 #필수 #디지털네트워크/QoS
## 정의
트래픽 정책처리(Policing)
- 설정된 대역폭을 초과하는 트래픽을 즉시 Drop 또는 Remark하는 트래픽 제어 기법
- 인바운드 트래픽에 적용되며 버퍼링 없이 초과 트래픽을 처리하는 메커니즘
## 키워드
* Token Bucket, CIR, PIR, CBS, PBS
* Drop, Remark, Exceed Action
* Inbound, Burst, Saw-tooth Pattern
## 암기법
* "폴드리" - Policing은 Drop/Remark 즉시
* "초과즉삭" - 초과 트래픽 즉시 삭제
## 구성요소/특징/유형
| 구분 | 설명 | 비고 |
|------|------|------|
| Token Bucket | 일정 속도로 토큰 생성, 패킷 전송 시 소비 | 버스트 허용 |
| CIR | Committed Information Rate, 보장 전송률 | 기본 대역폭 |
| PIR | Peak Information Rate, 최대 전송률 | 버스트 한계 |
| CBS | Committed Burst Size, 보장 버스트 크기 | Bc 버킷 |
| PBS | Peak Burst Size, 최대 버스트 크기 | Be 버킷 |
| Exceed Action | 초과 시 Drop/Remark/Transmit | 정책 정의 |
