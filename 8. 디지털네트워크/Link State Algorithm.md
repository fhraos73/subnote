#디지털네트워크 #라우팅 #알고리즘 #필수
## 정의
전체 토폴로지 기반 경로 결정, Link State
- 네트워크 전체 토폴로지 정보를 수집하여 최단 경로를 계산하는 라우팅 알고리즘
- Dijkstra 알고리즘 기반, OSPF/IS-IS 프로토콜에서 사용
## 키워드
* Dijkstra, OSPF, IS-IS, LSA, SPF, 토폴로지 데이터베이스, Area
## 암기법
* "다익오아엘" - Dijkstra, OSPF, IS-IS, Area, LSA
## 구성요소/특징/유형
| 구분 | 설명 | 비고 |
|------|------|------|
| LSA | Link State Advertisement | 링크 상태 광고 |
| LSDB | Link State Database | 토폴로지 DB |
| SPF | Shortest Path First | 최단 경로 계산 |
| Area | 계층적 영역 구분 | 확장성 확보 |
| Cost | 대역폭 기반 메트릭 | 홉 카운트보다 정확 |
### Distance Vector vs Link State 비교
| 구분 | Distance Vector | Link State |
|------|-----------------|------------|
| 알고리즘 | Bellman-Ford | Dijkstra |
| 정보 교환 | 라우팅 테이블 | 링크 상태 (LSA) |
| 네트워크 뷰 | 부분적 | 전체 |
| 수렴 속도 | 느림 | 빠름 |
| 메모리/CPU | 적음 | 많음 |
| 확장성 | 소규모 | 대규모 |
| 프로토콜 | RIP | OSPF, IS-IS |
