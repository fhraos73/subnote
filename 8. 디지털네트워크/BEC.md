#디지털네트워크 #오류제어 #ARQ #필수 #디지털네트워크/오류제어
## 정의
후진 오류 정정, BEC
- 수신측에서 오류 검출 후 송신측에 재전송을 요청하여 오류를 정정하는 방식
- ARQ(Automatic Repeat reQuest) 메커니즘 기반, 양방향 채널 필요
## 키워드
* ARQ(자동 반복 요청), ACK/NAK, 재전송
* CRC, 패리티, Stop-and-Wait, Go-Back-N, Selective Repeat
## 암기법
* "스고셀" - Stop-and-Wait, Go-Back-N, Selective Repeat
## 구성요소/특징/유형
| 구분 | 설명 | 비고 |
|------|------|------|
| 오류 검출 | CRC, 패리티, 체크섬 | 검출만 수행 |
| ACK | 정상 수신 확인 | Acknowledgement |
| NAK | 오류 발생 통보 | Negative ACK |
| Stop-and-Wait | 1프레임 전송 후 ACK 대기 | 단순, 비효율 |
| Go-Back-N | 오류 프레임부터 재전송 | 윈도우 기반 |
| Selective Repeat | 오류 프레임만 재전송 | 효율적, 복잡 |
## 구성도
```
[ARQ 유형 비교]

1. Stop-and-Wait ARQ
송신: [F1]──────────────▶ 대기 ──────────────▶[F2]──▶
수신:      ◀──[ACK1]────        ◀──[ACK2]────

2. Go-Back-N ARQ (윈도우=4)
송신: [F1][F2][F3][F4]──▶ [F3][F4][F5][F6]──▶
수신:      ◀──[NAK3]──── (F3부터 재전송)

3. Selective Repeat ARQ
송신: [F1][F2][F3][F4]──▶ [F3]만 재전송──▶
수신:      ◀──[NAK3]──── (F3만 재전송)
```
