#디지털네트워크 #HTTP #QUIC #옵션
## 정의
QUIC 기반 차세대 웹 프로토콜, HTTP/3
- UDP 기반 QUIC 프로토콜 위에서 동작하는 차세대 HTTP 표준
- Google이 개발한 QUIC을 IETF가 표준화하여 2022년 RFC 9114로 발표
## 키워드
* QUIC, UDP, 0-RTT, 멀티플렉싱, HOL Blocking 해결, TLS 1.3 내장
## 암기법
* 큐제멀영: QUIC기반, 0-RTT(제로RTT), 멀티플렉싱, 영속연결
* HTTP버전: 1.0(단순)→1.1(지속)→2.0(멀티)→3.0(QUIC)
## HTTP 버전별 비교
| 구분 | HTTP/1.1 | HTTP/2.0 | HTTP/3.0 |
|------|----------|----------|----------|
| 전송계층 | TCP | TCP | UDP(QUIC) |
| 연결방식 | Keep-Alive | 멀티플렉싱 | 멀티플렉싱 |
| HOL Blocking | 발생 | TCP레벨 발생 | 해결 |
| 핸드쉐이크 | 3-Way | 3-Way | 0-RTT/1-RTT |
| 암호화 | 선택(TLS) | 선택(TLS) | 필수(TLS 1.3) |
| 헤더압축 | 없음 | HPACK | QPACK |
## 특징
| 구분 | 설명 | 비고 |
|------|------|------|
| QUIC 기반 | UDP 위에 신뢰성 구현 | TCP 대체 |
| 0-RTT | 재연결 시 즉시 데이터 전송 | 지연시간 최소화 |
| 독립 스트림 | 스트림별 독립 전송 | HOL Blocking 해결 |
| 연결 마이그레이션 | IP 변경 시 연결 유지 | Connection ID 활용 |
| 내장 암호화 | TLS 1.3 기본 탑재 | 보안 강화 |
## 구성도
```
[HTTP 버전별 비교]

HTTP/1.1          HTTP/2.0           HTTP/3.0
┌────────┐       ┌────────┐        ┌────────┐
│ HTTP   │       │ HTTP/2 │        │ HTTP/3 │
├────────┤       ├────────┤        ├────────┤
│  TLS   │       │  TLS   │        │  QUIC  │←TLS 1.3 내장
├────────┤       ├────────┤        ├────────┤
│  TCP   │       │  TCP   │        │  UDP   │
├────────┤       ├────────┤        ├────────┤
│   IP   │       │   IP   │        │   IP   │
└────────┘       └────────┘        └────────┘

[HOL Blocking 해결]
HTTP/2 (TCP)                HTTP/3 (QUIC)
Stream1 ─┬─ TCP ─┬─        Stream1 ───────────▶
Stream2 ─┤      ├─ 손실    Stream2 ───────────▶
Stream3 ─┘      └─ 전체대기 Stream3 ──X(손실)──▶
                           (다른 스트림 영향 없음)
```
