#디지털네트워크 #오류검출 #데이터링크 #옵션 #디지털네트워크/오류제어
## 정의
다항식 기반 오류 검출, CRC
- 송신 데이터에 다항식 나눗셈을 적용하여 검사 비트(FCS)를 생성하는 오류 검출 기법
- 데이터링크 계층에서 프레임 오류 검출에 널리 사용되는 효율적인 방식
## 키워드
* 생성 다항식(Generator Polynomial), FCS(Frame Check Sequence), 모듈로-2 연산, CRC-16, CRC-32
## 암기법
* "생모프검" - 생성다항식, 모듈로-2, 프레임, 검사비트
## 구성요소/특징/유형
| 구분 | 설명 | 비고 |
|------|------|------|
| 생성 다항식 | 오류 검출용 다항식 (예: CRC-32) | 표준화됨 |
| FCS | Frame Check Sequence, 검사 비트 | 프레임 끝에 부착 |
| 모듈로-2 연산 | XOR 기반 나눗셈 | 캐리 없음 |
| CRC-16 | 16비트 검사, HDLC/SDLC | 2바이트 |
| CRC-32 | 32비트 검사, Ethernet/ZIP | 4바이트 |
### CRC 유형별 비교
| 유형 | 다항식 차수 | 적용 분야 |
|------|-------------|-----------|
| CRC-8 | 8비트 | ATM 헤더 |
| CRC-16 | 16비트 | HDLC, USB |
| CRC-32 | 32비트 | Ethernet, ZIP |
| CRC-CCITT | 16비트 | X.25, ISDN |

```
│   │   M(x)      │    │ (n비트)     │    │  G(x)로     │    │
│   └─────────────┘    └─────────────┘    └──────┬──────┘    │
│                                                 │           │
│                                                 ↓           │
│                                          ┌─────────────┐    │
│                                          │  나머지=FCS │    │

│   ┌─────────────────────────────────────────────┘           │
│   │                                                         │
│   ↓                                                         │

│   │    전송 프레임: M(x) + FCS    │ ────────────→ 전송      │
│   └───────────────────────────────┘                         │
│                                                             │
│   [수신측]                                                   │
│   ┌───────────────────────────────┐    ┌─────────────┐      │
│   │   수신 프레임: M(x) + FCS     │ →  │ G(x)로 나눗셈│      │

│                                               │             │

```
