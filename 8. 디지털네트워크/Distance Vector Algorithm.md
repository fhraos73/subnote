#디지털네트워크 #라우팅 #알고리즘 #필수
## 정의
거리 기반 경로 결정, Distance Vector
- 인접 라우터와 라우팅 테이블을 교환하여 목적지까지의 거리(홉 수)와 방향을 결정하는 라우팅 알고리즘
- Bellman-Ford 알고리즘 기반, RIP 프로토콜에서 사용
## 키워드
* Bellman-Ford, RIP, 홉 카운트, 라우팅 테이블, Count to Infinity, Split Horizon
## 암기법
* "벨립홉스카" - Bellman-Ford, RIP, 홉카운트, Split Horizon, Count to Infinity
## 구성요소/특징/유형
| 구분 | 설명 | 비고 |
|------|------|------|
| 거리 메트릭 | 홉 카운트 (최대 15) | 16 = 무한대 |
| 업데이트 주기 | 30초마다 전체 테이블 교환 | 대역폭 소모 |
| 수렴 시간 | 느림 (Count to Infinity) | 단점 |
| 구현 | 단순 | 소규모 네트워크 적합 |
### Distance Vector 문제점과 해결책
| 문제점 | 설명 | 해결책 |
|--------|------|--------|
| Count to Infinity | 무한 루프 발생 | 최대 홉 제한 (15) |
| Slow Convergence | 수렴 시간 김 | Triggered Update |
| Routing Loop | 잘못된 경로 순환 | Split Horizon |
| Hold-down | 불안정한 경로 | Hold-down Timer |
## 구성도
```
┌─────────────────────────────────────────────────────────────┐
│                Distance Vector 동작 원리                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [네트워크 토폴로지]                                         │
│       A ──(1)── B ──(1)── C ──(1)── D                       │
│                                                             │
│  [라우터 A의 라우팅 테이블 구축 과정]                         │
│                                                             │
│  초기 상태:                                                  │
│  ┌──────────────────────────────────┐                       │
│  │ 목적지 │ 거리 │ 다음 홉 │                                │
│  │   A    │  0   │   -    │                                │
│  │   B    │  1   │   B    │                                │
│  └──────────────────────────────────┘                       │
│                                                             │
│  B로부터 업데이트 수신 후:                                   │
│  ┌──────────────────────────────────┐                       │
│  │ 목적지 │ 거리 │ 다음 홉 │                                │
│  │   A    │  0   │   -    │                                │
│  │   B    │  1   │   B    │                                │
│  │   C    │  2   │   B    │ ← B의 C까지 거리(1) + 1        │
│  │   D    │  3   │   B    │ ← B의 D까지 거리(2) + 1        │
│  └──────────────────────────────────┘                       │
│                                                             │
│  [Bellman-Ford 공식]                                        │
│  D(x,y) = min { c(x,v) + D(v,y) }  (모든 이웃 v에 대해)     │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  [Split Horizon - 루프 방지]                                 │
│                                                             │
│       A ←──── B ←──── C                                     │
│                                                             │
│  • B는 A로부터 받은 경로 정보를 A에게 다시 전송하지 않음      │
│  • Poison Reverse: 해당 경로를 무한대(16)로 광고             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
