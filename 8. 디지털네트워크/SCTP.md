#디지털네트워크 #SCTP #전송계층 #옵션
## 정의
메시지 지향 신뢰성 전송 프로토콜, SCTP
- Stream Control Transmission Protocol, TCP와 UDP의 장점을 결합한 전송 계층 프로토콜
- 멀티호밍, 멀티스트리밍을 지원하는 메시지 기반 신뢰성 전송 프로토콜 (RFC 4960)
## 키워드
* 멀티호밍, 멀티스트리밍, Association, Chunk, 4-Way Handshake, 메시지경계
## 암기법
* SCTP 특징: 멀멀메신(멀티호밍, 멀티스트리밍, 메시지경계, 신뢰성)
* TCP+UDP: 신뢰(TCP) + 메시지경계(UDP) + 멀티호밍/스트리밍(SCTP 고유)
## TCP vs UDP vs SCTP 비교
| 구분 | TCP | UDP | SCTP |
|------|-----|-----|------|
| 연결 방식 | 연결지향 | 비연결 | 연결지향 |
| 신뢰성 | 보장 | 미보장 | 보장 |
| 메시지 경계 | 미보존 | 보존 | 보존 |
| 멀티호밍 | 미지원 | 미지원 | 지원 |
| 멀티스트리밍 | 미지원 | 미지원 | 지원 |
| HOL Blocking | 발생 | 없음 | 스트림별 독립 |
| 핸드쉐이크 | 3-Way | 없음 | 4-Way |
## 주요 특징
| 특징 | 설명 | 장점 |
|------|------|------|
| 멀티호밍 | 다중 IP 주소 지원 | 장애 시 자동 전환 |
| 멀티스트리밍 | 다중 스트림 전송 | HOL Blocking 해결 |
| 메시지 경계 보존 | 메시지 단위 전송 | 애플리케이션 단순화 |
| 4-Way Handshake | Cookie 기반 연결 | SYN Flooding 방지 |
| Chunk 기반 | 여러 메시지 묶음 전송 | 효율적 전송 |
## 구성도
```
[SCTP 4-Way Handshake]
  Client                           Server
    │                                │
    │──── INIT ─────────────────────▶│
    │     (요청 태그, 스트림 수)       │
    │                                │
    │◀─── INIT-ACK ──────────────────│
    │     (Cookie 포함)               │
    │                                │
    │──── COOKIE-ECHO ──────────────▶│
    │     (Cookie 반환)               │
    │                                │
    │◀─── COOKIE-ACK ────────────────│
    │                                │
    │◀════ Association 수립 ═════════▶│

[멀티호밍 (Multi-homing)]
┌─────────────┐                    ┌─────────────┐
│   Host A    │                    │   Host B    │
│  ┌───────┐  │    Primary Path    │  ┌───────┐  │
│  │IP1    │──┼────────────────────┼──│IP3    │  │
│  └───────┘  │                    │  └───────┘  │
│  ┌───────┐  │   Secondary Path   │  ┌───────┐  │
│  │IP2    │──┼─ ─ ─ ─ ─ ─ ─ ─ ─ ─┼──│IP4    │  │
│  └───────┘  │   (장애 시 전환)    │  └───────┘  │
└─────────────┘                    └─────────────┘

[멀티스트리밍 (Multi-streaming)]
┌─────────────────────────────────────────────────┐
│              SCTP Association                   │
├─────────────────────────────────────────────────┤
│  Stream 0: [Msg1] → [Msg2] → [Msg3]            │
│  Stream 1: [Msg4] → [Msg5] → [Msg6]            │
│  Stream 2: [Msg7] → [Msg8] → [Msg9]            │
└─────────────────────────────────────────────────┘
  ※ 각 스트림 독립적 → HOL Blocking 해결

[SCTP 활용 분야]
┌───────────────┬─────────────────────────────────┐
│ SS7 Signaling │ 통신망 시그널링 (SIGTRAN)       │
│ Diameter      │ 차세대 AAA 프로토콜             │
│ WebRTC        │ 데이터 채널 전송                │
│ 5G/LTE        │ 이동통신 제어 평면              │
└───────────────┴─────────────────────────────────┘
```
